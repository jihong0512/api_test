## 系统简介

API接口智能测试系统是一个基于大模型的智能API测试平台，主要功能包括：

- **智能文档解析**：自动解析OpenAPI/Swagger文档，提取接口信息
- **智能测试用例生成**：基于大模型自动生成测试用例
- **接口依赖分析**：自动分析接口间的依赖关系和业务逻辑
- **数据库连接分析**：连接业务数据库，分析数据结构，辅助测试数据生成
- **测试执行与报告**：执行测试用例，生成详细的测试报告
- **知识图谱**：可视化展示接口和数据库的关系网络

## 快速开始

### 1. 登录系统

1. 访问系统地址：http://localhost:3006
2. 首次使用请在登录页点击“注册”创建账号后登录（系统不预置默认账号）
   - 也可通过后端接口创建用户：在 http://localhost:8004/docs 使用 /api/auth/register 接口

### 2. 创建项目

1. 登录后，点击左侧菜单的 **"项目管理"**
2. 点击 **"创建项目"** 按钮
3. 填写项目信息：
   - 项目名称
   - 项目描述
   - 基础URL（可选）
4. 点击 **"确定"** 创建项目

### 3. 上传API文档（推荐方式）

1. 在项目详情页面，点击 **"文档管理"**
2. 点击 **"上传文档"** 按钮
3. 选择OpenAPI/Swagger文档文件（支持JSON/YAML格式）
4. 系统会自动解析文档并提取接口信息

### 4. 手动添加接口（可选）

1. 在项目详情页面，点击 **"接口管理"**
2. 点击 **"添加接口"** 按钮
3. 填写接口信息：
   - 接口名称
   - HTTP方法（GET/POST/PUT/DELETE等）
   - 接口路径
   - 请求参数
   - 响应格式
4. 点击 **"保存"**

### 5. 配置数据库连接（可选，用于辅助测试数据生成）

1. 在项目详情页面，点击 **"数据库信息"**
2. 点击 **"创建数据库连接"** 按钮
3. 填写数据库连接信息：
   - 数据库类型：MySQL
   - 主机地址
   - 端口号
   - 数据库名（如：xiaojin）
   - 用户名
   - 密码
4. 点击 **"测试连接"** 验证连接
5. 点击 **"确定"** 保存连接

### 6. 生成测试用例

1. 在 **"接口管理"** 页面，选择要生成测试用例的接口
2. 点击 **"生成测试用例"** 按钮
3. 系统会使用大模型自动生成测试用例
4. 生成完成后，可以在 **"测试用例"** 页面查看

### 7. 配置测试环境

1. 在项目详情页面，点击 **"测试环境"**
2. 点击 **"添加环境"** 按钮
3. 填写环境信息：
   - 环境名称（如：开发环境、测试环境）
   - 基础URL
   - 认证信息（如：API Key、Token等）
4. 点击 **"保存"**

### 8. 执行测试

1. 在 **"测试用例"** 页面，选择要执行的测试用例
2. 点击 **"执行测试"** 按钮
3. 选择测试环境
4. 系统会自动执行测试并显示结果

### 9. 查看测试报告

1. 在 **"测试结果"** 页面查看测试执行结果
2. 点击具体的测试结果查看详细信息
3. 查看失败用例的错误信息和修复建议

## 核心功能使用指南

### 1. 文档管理

**功能说明**：上传和管理API文档，系统会自动解析文档中的接口信息。

**操作步骤**：
1. 进入 **"文档管理"** 页面
2. 点击 **"上传文档"**
3. 选择OpenAPI/Swagger文档（支持.json、.yaml、.yml格式）
4. 系统会自动解析并提取接口信息

**注意事项**：
- 支持OpenAPI 3.0和Swagger 2.0格式
- 文档解析可能需要几分钟时间
- 解析完成后，接口会自动添加到"接口管理"中

### 2. 接口管理

**功能说明**：查看和管理项目中的所有API接口。

**主要操作**：
- **查看接口详情**：点击接口名称查看详细信息
- **编辑接口**：点击"编辑"按钮修改接口信息
- **删除接口**：点击"删除"按钮移除接口
- **生成测试用例**：选择接口后点击"生成测试用例"

### 3. 接口依赖分析

**功能说明**：自动分析接口间的依赖关系，识别业务逻辑流程。

**操作步骤**：
1. 进入 **"接口依赖"** 页面
2. 系统会自动分析项目中的所有接口
3. 查看依赖关系图和业务流程图

**依赖类型**：
- **认证依赖**：需要先调用登录接口获取token
- **数据依赖**：需要先调用创建接口获取ID
- **业务依赖**：按照业务逻辑顺序调用接口

### 4. 测试用例管理

**功能说明**：查看和管理测试用例，支持手动编辑和自动生成。

**主要操作**：
- **查看测试用例**：查看用例的请求数据、断言规则等
- **编辑测试用例**：修改测试数据和断言规则
- **执行测试用例**：单独执行某个测试用例
- **批量执行**：选择多个测试用例批量执行

### 5. 数据库连接管理

**功能说明**：连接业务数据库，用于分析数据结构，辅助生成测试数据。

**操作步骤**：
1. 进入 **"数据库信息"** 页面
2. 点击 **"创建数据库连接"**
3. 填写数据库连接信息
4. 点击 **"测试连接"** 验证
5. 连接成功后，系统会自动分析数据库结构

**功能特性**：
- 自动分析表结构和字段类型
- 识别表之间的外键关系
- 生成数据库关系图
- 辅助测试数据生成

### 6. 测试环境管理

**功能说明**：管理不同的测试环境（开发、测试、生产等）。

**配置项**：
- **环境名称**：用于区分不同环境
- **基础URL**：API的基础地址
- **认证信息**：API Key、Token等认证凭证
- **请求头**：全局请求头配置

### 7. 测试执行

**功能说明**：执行测试用例，收集测试结果。

**执行方式**：
- **单个执行**：执行单个测试用例
- **批量执行**：执行多个测试用例
- **套件执行**：执行整个测试套件
- **流程执行**：按照业务逻辑流程执行

**执行结果**：
- 请求和响应详情
- 断言结果
- 执行时间
- 错误信息（如有）

### 8. 测试报告

**功能说明**：查看详细的测试报告和分析结果。

**报告内容**：
- 测试执行概览（总数、通过数、失败数）
- 失败用例详情和错误分析
- 性能数据（响应时间、吞吐量等）
- 测试趋势分析

### 9. 知识图谱

**功能说明**：可视化展示接口、数据库、实体之间的关系网络。

**查看方式**：
1. 进入 **"知识图谱"** 页面
2. 查看接口、数据库表、实体之间的关系图
3. 点击节点查看详细信息

## 配置指南

### 一、配置大模型API

系统使用大模型进行智能分析，需要配置大模型的API密钥和相关信息。

#### 1.1 配置DeepSeek API（主要模型）

**文件位置**：`backend/app/config.py`

**修改位置**：第40-42行

**原始代码**：
```python
# DeepSeek API配置
DEEPSEEK_API_KEY: str = "sk-abebebd1bc2b404fa0556d46912a379e"
DEEPSEEK_BASE_URL: str = "https://api.deepseek.com"
DEEPSEEK_MODEL: str = "deepseek-chat"
```

**修改方法**：
1. 打开文件 `backend/app/config.py`
2. 找到第40行，修改 `DEEPSEEK_API_KEY` 的值，替换为您的DeepSeek API密钥
3. 找到第41行，如果需要使用不同的API地址，修改 `DEEPSEEK_BASE_URL`
4. 找到第42行，如果需要使用不同的模型，修改 `DEEPSEEK_MODEL`（例如：`deepseek-chat`、`deepseek-reasoner`等）

**示例**：
```python
# DeepSeek API配置
DEEPSEEK_API_KEY: str = "sk-your-api-key-here"
DEEPSEEK_BASE_URL: str = "https://api.deepseek.com"
DEEPSEEK_MODEL: str = "deepseek-chat"
```

#### 1.2 配置通义千问API（用于Embedding和Rerank）

**修改位置**：第45-47行

#### 1.3 配置Docker环境变量（如果使用Docker部署）

**文件位置**：`docker-compose.yml`

**修改位置**：第137-141行（backend服务）和第191-195行（celery_worker服务）

**原始代码**（backend服务，第137-141行）：
```yaml
- DEEPSEEK_API_KEY=sk-abebebd1bc2b404fa0556d46912a379e
- DEEPSEEK_BASE_URL=https://api.deepseek.com
- QWEN_API_KEY=sk-5233a3a4b1a24426b6846a432794bbe2
```

**修改方法**：

3. 修改 `DEEPSEEK_API_KEY`、`DEEPSEEK_BASE_URL`、`QWEN_API_KEY`、`QWEN_BASE_URL`、`QWEN_MODEL` 的值
4. **注意**：如果需要修改 `DEEPSEEK_MODEL`，需要添加环境变量（当前docker-compose.yml中没有此配置，会使用config.py中的默认值）
5. 同样修改celery_worker服务的environment部分（第191-195行）

**示例**：
```yaml
- DEEPSEEK_API_KEY=sk-your-api-key-here
- DEEPSEEK_BASE_URL=https://api.deepseek.com
- DEEPSEEK_MODEL=deepseek-chat  # 如果需要修改模型，添加此行
- QWEN_API_KEY=sk-your-qwen-api-key-here
- QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
- QWEN_MODEL=qwen-vl-plus
```

**重要提示**：
- 修改Docker配置后，需要重启服务才能生效
- 重启命令：`docker compose restart backend celery_worker`
- 或者重新部署：`docker compose down && docker compose up -d`

### 二、配置系统数据库（api_test数据库）

系统自身使用的数据库配置，用于存储项目、接口、测试用例等数据。

#### 2.1 修改Python配置文件

**文件位置**：`backend/app/config.py`

**修改位置**：第11-15行

**原始代码**：
```python
# MySQL配置
MYSQL_HOST: str = "localhost"
MYSQL_PORT: int = 3306
MYSQL_USER: str = "root"
MYSQL_PASSWORD: str = "123456"
MYSQL_DATABASE: str = "api_test"
```

**修改方法**：
1. 打开文件 `backend/app/config.py`
2. 找到第11行，修改 `MYSQL_HOST` 为您的MySQL服务器地址
3. 找到第12行，修改 `MYSQL_PORT` 为您的MySQL端口（默认3306）
4. 找到第13行，修改 `MYSQL_USER` 为您的MySQL用户名
5. 找到第14行，修改 `MYSQL_PASSWORD` 为您的MySQL密码
6. 找到第15行，修改 `MYSQL_DATABASE` 为您的数据库名（默认api_test）

**示例**：
```python
# MySQL配置
MYSQL_HOST: str = "192.168.1.100"
MYSQL_PORT: int = 3306
MYSQL_USER: str = "testuser"
MYSQL_PASSWORD: str = "your-password"
MYSQL_DATABASE: str = "api_test"
```

#### 2.2 修改Docker环境变量（如果使用Docker部署）

**文件位置**：`docker-compose.yml`

**修改位置**：第120-124行（backend服务）和第174-178行（celery_worker服务）

**原始代码**（backend服务，第120-124行）：
```yaml
- MYSQL_HOST=mysql
- MYSQL_PORT=3306
- MYSQL_USER=root
- MYSQL_PASSWORD=123456
- MYSQL_DATABASE=api_test
```

**修改方法**：
1. 打开文件 `docker-compose.yml`

3. 修改 `MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_DATABASE` 的值
4. 同样修改celery_worker服务的environment部分（第174-178行）

- 如果MySQL在外部服务器，`MYSQL_HOST` 应该使用IP地址或域名

```yaml
- MYSQL_PASSWORD=your-password

```

**示例**（MySQL在外部服务器）：

### 三、配置业务数据库连接（xiaojin数据库）

**重要说明**：xiaojin数据库是业务数据库，不是系统数据库。这个数据库连接是通过系统界面配置的，用于辅助测试数据生成和接口依赖分析。

#### 3.1 通过系统界面配置（推荐方式）

**操作步骤**：
1. 登录系统，进入项目详情页面
2. 点击左侧菜单的 **"数据库信息"**
3. 点击 **"创建数据库连接"** 按钮
4. 填写数据库连接信息：
   - **数据库类型**：选择 MySQL
   - **主机地址**：填写数据库服务器地址（如：120.92.88.231）
   - **端口号**：填写数据库端口（如：63694）
   - **数据库名**：填写数据库名称（如：xiaojin）
   - **用户名**：填写数据库用户名（如：admin）
   - **密码**：填写数据库密码（如：8S!5q&jJHQkn6Mn%jBjwA）
5. 点击 **"测试连接"** 验证连接是否成功
6. 连接成功后，点击 **"确定"** 保存

**配置示例**：
```
数据库类型：MySQL
主机地址：120.92.88.231
端口号：6369
数据库名：xiaojin
用户名：admin
密码：8S!5q&jJHQkn6M
```

#### 3.2 数据库连接信息存储位置

数据库连接信息存储在系统数据库的 `db_connections` 表中，每个项目可以有多个数据库连接。

**查看连接信息**：
- 通过系统界面：在"数据库信息"页面查看所有连接
- 通过数据库：查询 `api_test` 数据库的 `db_connections` 表

**注意事项**：
- 数据库连接信息是加密存储的，密码不会明文显示
- 每个项目可以配置多个数据库连接
- 连接信息与项目关联，不同项目可以使用不同的数据库

## 常见问题

### Q1: 如何获取DeepSeek API密钥？

**A**: 
1. 访问 DeepSeek 官网：https://www.deepseek.com
2. 注册/登录账号
3. 进入控制台，创建API密钥
4. 复制API密钥，配置到系统中

### Q2: 如何获取通义千问API密钥？

**A**: 
1. 访问阿里云DashScope：https://dashscope.aliyuncs.com
2. 注册/登录账号
3. 创建API密钥
4. 复制API密钥，配置到系统中

### Q3: 修改配置后不生效怎么办？

**A**: 
1. 确认配置文件已保存
2. 如果使用Docker，需要重启服务：
   ```bash
   docker compose restart backend celery_worker
   ```
3. 检查日志是否有错误：
   ```bash
   docker compose logs backend
   ```

### Q4: 如何测试数据库连接是否成功？

**A**: 
1. 在系统界面的"数据库信息"页面，点击"测试连接"按钮
2. 如果连接成功，会显示"连接成功"提示
3. 如果连接失败，会显示具体的错误信息

### Q5: 系统支持哪些数据库类型？

**A**: 
- 系统自身数据库：MySQL
- 业务数据库连接：目前支持MySQL，后续可能支持PostgreSQL、MongoDB等

### Q6: 如何查看系统日志？

**A**: 
1. **Docker部署**：
   ```bash
   # 查看所有服务日志
   docker compose logs -f
   
   # 查看后端服务日志
   docker compose logs -f backend
   
   # 查看Celery Worker日志
   docker compose logs -f celery_worker
   ```

2. **直接运行**：
   - 查看后端日志文件（通常在项目根目录的logs文件夹）

### Q7: 测试用例生成失败怎么办？

**A**: 
1. 检查大模型API配置是否正确
2. 检查API密钥是否有效
3. 查看后端日志，确认具体错误信息
4. 确认网络连接正常，可以访问大模型API

### Q8: 如何修改默认的模型？

**A**: 
1. 修改 `backend/app/config.py` 文件
2. 修改 `DEEPSEEK_MODEL` 或 `QWEN_MODEL` 的值
3. 重启服务

**支持的模型**：
- DeepSeek: `deepseek-chat`、`deepseek-reasoner`等
- 通义千问: `qwen-vl-plus`、`qwen-turbo`、`qwen-max`等
