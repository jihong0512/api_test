# 文档上传支持的文件格式说明

## 概述

apitest平台支持多种文件格式的API接口文档上传，系统会自动识别文件类型并提取接口信息。

## 支持的文件类型

### 1. Swagger/OpenAPI 格式 ⭐（推荐）

**文件类型：** `swagger`、`yaml`、`yml`、`json`

**识别方式：**
- 文件名包含 `swagger` 或 `openapi` 关键词
- 文件内容包含 `openapi` 或 `swagger` 字段

**文件格式：**
- **YAML格式** (`.yaml`, `.yml`)：OpenAPI 3.0.3 规范
- **JSON格式** (`.json`)：OpenAPI 3.0.3 规范

**必需字段：**
```yaml
openapi: 3.0.3  # 或 swagger: 2.0
info:
  title: API文档标题
  version: 1.0.0
paths:
  /api/endpoint:
    get:
      summary: 接口名称
      description: 接口描述
      parameters: []
      responses: {}
```

**示例：**
```yaml
openapi: 3.0.3
info:
  title: 测试API文档
  version: 1.0.0
servers:
  - url: http://localhost:8000
paths:
  /api/users:
    get:
      summary: 获取用户列表
      parameters:
        - name: page
          in: query
          schema:
            type: integer
      responses:
        '200':
          description: 成功
```

---

### 2. Postman Collection 格式

**文件类型：** `postman`、`json`

**识别方式：**
- 文件名包含 `postman` 关键词
- 文件扩展名为 `.postman_collection.json`

**文件格式：** Postman Collection v2.1 JSON格式

**必需字段：**
```json
{
  "info": {
    "name": "Collection名称"
  },
  "item": [
    {
      "name": "接口名称",
      "request": {
        "method": "GET",
        "url": {
          "raw": "http://api.example.com/users",
          "host": ["api", "example", "com"],
          "path": ["users"]
        },
        "header": [
          {
            "key": "Content-Type",
            "value": "application/json"
          }
        ],
        "body": {}
      }
    }
  ]
}
```

**示例：**
```json
{
  "info": {
    "name": "我的API集合"
  },
  "item": [
    {
      "name": "获取用户信息",
      "request": {
        "method": "GET",
        "url": {
          "raw": "https://api.example.com/users/123"
        },
        "header": [],
        "body": {}
      }
    }
  ]
}
```

---

### 3. Apifox 格式

**文件类型：** `apifox`、`json`

**识别方式：**
- 文件名包含 `apifox` 关键词

**文件格式：** Apifox导出的JSON格式

**必需字段：**
```json
{
  "apis": [
    {
      "name": "接口名称",
      "method": "GET",
      "path": "/api/users",
      "headers": {},
      "params": {},
      "requestBody": {},
      "response": {}
    }
  ]
}
```

**或：**
```json
{
  "data": {
    "apis": [...]
  }
}
```

**示例：**
```json
{
  "apis": [
    {
      "name": "创建用户",
      "method": "POST",
      "path": "/api/users",
      "headers": {
        "Content-Type": "application/json"
      },
      "requestBody": {
        "name": "string",
        "email": "string"
      },
      "response": {
        "id": 1,
        "name": "string"
      }
    }
  ]
}
```

---

### 4. JSON 请求日志格式

**文件类型：** `json`

**识别方式：**
- JSON数组格式，每个元素包含 `url`、`method` 等字段

**文件格式：** JSON数组，每个元素是一个API请求记录

**必需字段：**
```json
[
  {
    "title": "接口名称（可选）",
    "url": "http://api.example.com/users",
    "method": "POST",
    "headers": {},
    "request_body": {},
    "response_body": {},
    "base_url": "http://api.example.com",
    "service": "用户服务"
  }
]
```

**支持的字段：**
- `url` / `path` / `endpoint`：接口URL
- `method`：HTTP方法（GET/POST/PUT/DELETE等）
- `headers`：请求头（对象或JSON字符串）
- `request_body` / `body`：请求体（对象或JSON字符串）
- `response_body` / `response`：响应体（对象或JSON字符串）
- `base_url`：基础URL
- `service`：服务名称
- `title` / `name`：接口名称
- `description` / `desc`：接口描述

**示例：**
```json
[
  {
    "title": "用户登录",
    "url": "https://api.example.com/auth/login",
    "method": "POST",
    "headers": {
      "Content-Type": "application/json"
    },
    "request_body": {
      "username": "test",
      "password": "123456"
    },
    "response_body": {
      "token": "eyJhbGci...",
      "user_id": 1
    },
    "base_url": "https://api.example.com",
    "service": "认证服务"
  }
]
```

**特殊格式：**
- 如果JSON对象包含 `interfaces`、`data`、`items` 字段，系统会自动提取数组内容

### 5. JMeter JMX 格式

**文件类型：** `jmx`

**识别方式：**
- 文件扩展名为 `.jmx`

**文件格式：** JMeter测试计划XML格式

**必需结构：**
```xml
<jmeterTestPlan>
  <hashTree>
    <TestPlan>
      <hashTree>
        <HTTPSamplerProxy>
          <stringProp name="HTTPSampler.domain">api.example.com</stringProp>
          <stringProp name="HTTPSampler.path">/users</stringProp>
          <stringProp name="HTTPSampler.method">GET</stringProp>
        </HTTPSamplerProxy>
      </hashTree>
    </TestPlan>
  </hashTree>
</jmeterTestPlan>
```

**提取的信息：**
- `HTTPSampler.domain`：域名
- `HTTPSampler.path`：路径
- `HTTPSampler.method`：HTTP方法
- `HTTPSampler.port`：端口号

---

### 6. 文本文件格式

**文件类型：** `txt`、`md`

**识别方式：**
- 文件扩展名为 `.txt` 或 `.md`

**文件格式：** 纯文本或Markdown格式

**内容要求：**
- 包含API接口描述信息
- 可以包含URL、方法、参数等文本描述
- 系统会尝试从文本中提取接口信息

**示例（Markdown）：**
```markdown
# API接口文档

## 1. 获取用户列表
- **URL:** GET /api/users
- **参数:** page, size
- **描述:** 分页获取用户列表

## 2. 创建用户
- **URL:** POST /api/users
- **请求体:** 
  ```json
  {
    "name": "string",
    "email": "string"
  }
  ```
```

---

### 7. 其他格式（文本提取）

**文件类型：** `pdf`、`docx`、`xlsx`、`csv`、`xml`、`xmind`

**识别方式：**
- 根据文件扩展名自动识别

**文件格式：**
- **PDF** (`.pdf`)：包含API文档的PDF文件
- **Word** (`.docx`, `.doc`)：包含API文档的Word文件
- **Excel** (`.xlsx`, `.xls`)：包含接口列表的Excel表格
- **CSV** (`.csv`)：包含接口信息的CSV表格
- **XML** (`.xml`)：XML格式的接口文档
- **XMind** (`.xmind`)：思维导图格式

**内容要求：**
- 包含API接口描述信息
- 系统会提取文本内容，并尝试识别接口信息
- 表格格式更易于提取结构化数据

**Excel示例：**
| 接口名称 | 方法 | URL | 参数 | 描述 |
|---------|------|-----|------|------|
| 获取用户 | GET | /api/users | page, size | 分页获取用户列表 |
| 创建用户 | POST | /api/users | name, email | 创建新用户 |

## 文件类型自动识别规则

系统按以下优先级识别文件类型：

1. **文件名关键词识别**（优先级最高）
   - 包含 `postman` → `postman`
   - 包含 `apifox` → `apifox`
   - 包含 `swagger` 或 `openapi` → `swagger`

2. **文件扩展名识别**
   - `.yaml`, `.yml` → `yaml`（如果包含openapi/swagger字段，自动识别为swagger）
   - `.json` → `json`（根据内容判断是否为Postman/Apifox/Swagger）
   - `.txt` → `txt`
   - `.md` → `markdown`
   - `.pdf` → `pdf`
   - `.docx`, `.doc` → `docx`
   - `.xlsx`, `.xls` → `excel`
   - `.csv` → `csv`
   - `.xml` → `xml`
   - `.jmx` → `jmx`
   - `.xmind` → `xmind`

3. **内容检测**
   - YAML文件如果包含 `openapi` 或 `swagger` 字段，自动按Swagger格式解析
   - JSON文件如果包含 `item` 字段（Postman格式），按Postman解析
   - JSON文件如果包含 `apis` 字段（Apifox格式），按Apifox解析

## 接口信息提取规则

系统会从文档中提取以下接口信息：

### 必需字段
- `name`：接口名称
- `method`：HTTP方法（GET/POST/PUT/DELETE/PATCH等）
- `url`：完整URL或路径

### 可选字段
- `base_url`：基础URL
- `path`：接口路径
- `headers`：请求头（对象）
- `params`：查询参数（对象）
- `body` / `request_body`：请求体（对象）
- `response_schema`：响应结构（对象）
- `description`：接口描述
- `tags`：标签列表
- `version`：版本信息
- `service`：服务名称

## 推荐格式

### 最佳实践（按推荐程度排序）

1. **Swagger/OpenAPI YAML格式** ⭐⭐⭐⭐⭐
   - 标准化程度高
   - 信息完整
   - 自动提取接口信息最准确

2. **Swagger/OpenAPI JSON格式** ⭐⭐⭐⭐⭐
   - 与YAML格式等价
   - 适合程序生成

3. **Postman Collection** ⭐⭐⭐⭐
   - 从Postman直接导出
   - 包含完整的请求信息

4. **JSON请求日志** ⭐⭐⭐
   - 适合从实际请求中收集
   - 包含真实请求和响应数据

5. **Apifox格式** ⭐⭐⭐
   - 从Apifox直接导出
   - 格式规范

6. **其他格式（PDF/Word/Excel等）** ⭐⭐
   - 需要人工编写文档
   - 提取准确度取决于文档格式规范程度

---

## 注意事项

1. **文件编码**：建议使用 UTF-8 编码
2. **文件大小**：建议单个文件不超过 50MB
3. **接口数量**：建议单个文档不超过 1000 个接口
4. **格式规范**：遵循标准格式规范可以提高提取准确度
5. **文件命名**：包含关键词（如 `openapi`、`postman`）有助于自动识别

## 常见问题

### Q: 我的文件格式不在列表中怎么办？
A: 可以尝试：
1. 转换为JSON格式（请求日志格式）
2. 转换为Markdown或文本格式
3. 使用Swagger/OpenAPI格式（推荐）

### Q: 为什么我的文档解析后没有接口？
A: 可能原因：
1. 文件格式不规范
2. 缺少必需的字段（url、method等）
3. 文件类型识别错误

### Q: 如何提高接口提取准确度？
A: 建议：
1. 使用标准格式（Swagger/OpenAPI）
2. 确保包含完整的接口信息
3. 使用规范的字段名称
