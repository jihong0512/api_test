# APITest 本地开发模式部署说明

## 部署架构

- **Docker 容器运行**：MySQL、Redis、Neo4j、MinIO（基础服务）
- **本地运行**：Backend、Frontend、Celery Worker（应用服务）

## 一、启动基础服务（Docker）

### 方式1：使用开发环境配置（推荐）

```bash
cd /Users/jihongqing/new_ai_workspace/apitest
docker-compose -f docker-compose.dev.yml up -d
```

### 方式2：手动启动基础服务

如果使用完整的 docker-compose.yml，可以只启动基础服务：

```bash
cd /Users/jihongqing/new_ai_workspace/apitest
docker-compose up -d mysql redis neo4j minio
```

### 验证基础服务

```bash
# 查看服务状态
docker-compose -f docker-compose.dev.yml ps

# 检查MySQL
docker-compose -f docker-compose.dev.yml exec mysql mysqladmin ping -h localhost -uroot -p123456

# 检查Redis
docker-compose -f docker-compose.dev.yml exec redis redis-cli ping

# 检查MinIO（浏览器访问）
# http://localhost:9006 (minioadmin/minioadmin123456)

# 检查Neo4j（浏览器访问）
# http://localhost:7475 (neo4j/123456789)
```

## 二、配置本地环境变量

### 创建后端环境变量文件（可选）

在 `backend` 目录下创建 `.env` 文件（如果不存在）：

```bash
cd /Users/jihongqing/new_ai_workspace/apitest/backend
```

创建 `.env` 文件：

```bash
# 复制示例文件
cp .env.example .env

# 或手动创建 .env 文件
```

`.env` 文件内容示例（参考 `backend/.env.example`）：

```env
# MySQL配置（连接到Docker中的MySQL）
# 注意：Docker容器映射端口为3309
MYSQL_HOST=localhost
MYSQL_PORT=3309
MYSQL_USER=root
MYSQL_PASSWORD=123456
MYSQL_DATABASE=api_test

# Redis配置（连接到Docker中的Redis）
# 注意：Docker容器映射端口为6382
REDIS_HOST=localhost
REDIS_PORT=6382

# Neo4j配置
# 注意：Bolt端口使用7688（避免与uitest的7687端口冲突）
NEO4J_URI=bolt://localhost:7688
NEO4J_USER=neo4j
NEO4J_PASSWORD=123456789

# MinIO配置（连接到Docker中的MinIO）
# 注意：Docker容器映射端口为9005
MINIO_ENDPOINT=localhost:9005
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin123456
```

**重要提示**：
- 如果没有创建 `.env` 文件，应用会使用 `app/config.py` 中的默认值
- **但是默认端口不匹配！** 必须通过环境变量覆盖：
  - MySQL: 默认3306 → 需要设置为3309
  - Redis: 默认6379 → 需要设置为6382
  - MinIO: 默认9000 → 需要设置为9005
  - Neo4j: 默认7687 → 需要设置为7688（避免与uitest的7687端口冲突）

推荐创建 `.env` 文件并设置正确的端口。

## 三、本地启动 Backend

### 1. 安装Python依赖

```bash
cd /Users/jihongqing/new_ai_workspace/apitest/backend

# 创建虚拟环境（推荐）
python3 -m venv venv
source venv/bin/activate  # Mac/Linux

# 安装依赖
pip install -r requirements.txt
```

### 2. 启动Backend服务

```bash
# 方式1：使用启动脚本（推荐）
python start_server.py

# 方式2：直接使用uvicorn
uvicorn main:app --host 0.0.0.0 --port 8004 --reload

# 方式3：使用环境变量指定端口
SERVER_PORT=8004 python start_server.py
```

Backend 将运行在：**http://localhost:8004**

验证：访问 http://localhost:8004/docs 查看API文档

## 四、本地启动 Celery Worker

### 启动Celery Worker

```bash
cd /Users/jihongqing/new_ai_workspace/apitest/backend

# 确保虚拟环境已激活
source venv/bin/activate  # 如果使用虚拟环境

# 设置环境变量（如果需要）
export REDIS_HOST=localhost
export REDIS_PORT=6382

# 启动Celery Worker
celery -A app.celery_app worker --loglevel=info --concurrency=4

# 或者在后台运行
celery -A app.celery_app worker --loglevel=info --concurrency=4 &
```

**注意**：Celery Worker 需要连接到 Redis，确保 Redis 服务已启动（端口6382）。

## 五、本地启动 Frontend

### 1. 安装Node.js依赖

```bash
cd /Users/jihongqing/new_ai_workspace/apitest/frontend

# 安装依赖
npm install
# 或使用yarn
yarn install
```

### 2. 配置前端环境变量

创建 `.env` 文件（可选，package.json中已配置proxy）：

```env
REACT_APP_API_URL=http://localhost:8004
PORT=3006
```

### 3. 启动前端服务

```bash
npm start
# 或
PORT=3006 npm start
```

Frontend 将运行在：**http://localhost:3006**

**注意**：前端会通过proxy自动转发API请求到后端（localhost:8004）。

## 六、完整启动流程

### 快速启动脚本

可以创建一个启动脚本 `start_local.sh`：

```bash
#!/bin/bash

# 启动基础服务
echo "启动基础服务（Docker）..."
cd /Users/jihongqing/new_ai_workspace/apitest
docker-compose -f docker-compose.dev.yml up -d

# 等待服务就绪
echo "等待基础服务就绪..."

# 启动后端（新终端）
echo "请在新的终端窗口运行后端："
echo "cd /Users/jihongqing/new_ai_workspace/apitest/backend"
echo "source venv/bin/activate  # 如果使用虚拟环境"
echo "python start_server.py"

# 启动Celery Worker（新终端）
echo "请在新的终端窗口运行Celery Worker："
echo "cd /Users/jihongqing/new_ai_workspace/apitest/backend"
echo "source venv/bin/activate  # 如果使用虚拟环境"
echo "celery -A app.celery_app worker --loglevel=info --concurrency=4"

# 启动前端（新终端）
echo "请在新的终端窗口运行前端："
echo "cd /Users/jihongqing/new_ai_workspace/apitest/frontend"
echo "npm start"
```

## 九、常见问题

### 1. 端口被占用

如果遇到端口被占用，可以：
- 检查端口占用：`lsof -i :端口号`
- 修改配置文件中的端口
- 停止占用端口的服务

### 2. 无法连接到数据库

- 确保 Docker 容器正在运行：`docker-compose -f docker-compose.dev.yml ps`
- 检查端口映射是否正确（3309, 6382, 7688, 9005）
- 检查环境变量配置是否正确

### 3. Celery Worker 无法启动

- 确保 Redis 服务正在运行
- 检查 Redis 连接配置（REDIS_HOST, REDIS_PORT）
- 查看 Celery 日志确认错误信息

### 4. 前端无法连接到后端

- 确保 Backend 服务正在运行（http://localhost:8004）
- 检查前端 proxy 配置（package.json）
- 检查浏览器控制台的错误信息

| 服务 | UITest | APITest (本地) |
|------|--------|----------------|
| MySQL | 3312 | 3309 |
| Redis | 6385 | 6382 |
| Neo4j | 7475/7688 | 7474/7687 |
| MinIO | 9008/9009 | 9005/9006 |
| Backend | 8007 | 8004 (本地) |
| Frontend | 3009 | 3006 (本地) |

## 十一、开发建议

1. **使用虚拟环境**：为 Python 项目创建独立的虚拟环境
2. **使用环境变量文件**：创建 `.env` 文件管理配置，不要提交到Git
3. **使用代码热重载**：Backend 和 Frontend 都支持热重载，修改代码后自动重启
4. **查看日志**：使用多个终端分别运行服务，方便查看日志
5. **使用 Docker Compose 管理基础服务**：便于统一管理和启动
