# 接口场景用例和接口测试用例说明

## 问题回答

### Q1: "接口场景用例"标签页中的"生成接口用例"按钮，生成的是接口场景用例还是接口测试用例？

**答案：生成的是接口场景用例**

**原因分析**：

1. **按钮位置**：按钮位于"接口场景用例"标签页（`activeTab === 'scenario'`）
2. **后端逻辑**：调用 `/api/test-cases/generate-by-module?case_type=pytest`
3. **生成的测试用例名称格式**（`test_cases.py` 第1124行）：
   ```python
   module_test_case_name = f"{module}_测试用例"
   ```
   - 如果 `module="场景"`，生成的名称就是 `"场景_测试用例"`
   - 名称包含"场景"，符合场景用例的筛选规则

4. **筛选规则**（`test_cases.py` 第247-253行）：
   ```python
   if is_scenario is not None:
       if is_scenario:
           # 只返回场景用例：名称包含"场景"的用例
           query = query.filter(TestCase.name.like('%场景%'))
       else:
           # 只返回普通接口用例：名称不包含"场景"的用例
           query = query.filter(~TestCase.name.like('%场景%'))
   ```

5. **结果**：
   - 生成的测试用例名称包含"场景"（例如："场景_测试用例"）
   - 会被筛选到"接口场景用例"标签页（`is_scenario=true`）
   - **不会出现在"接口测试用例"标签页**

**结论**：虽然按钮名称是"生成接口用例"，但由于名称格式包含模块名（可能是"场景"），实际生成的是接口场景用例。

### Q2: "接口测试用例"标签页下面显示的是接口用例还是接口场景用例？

**答案：显示的是接口测试用例（接口用例）**

**原因分析**：

1. **筛选条件**（`TestCases.js` 第196-199行）：
   ```javascript
   } else if (activeTab === 'interface') {
     // 接口测试用例：pytest类型，但不是场景用例（普通接口用例）
     caseType = 'pytest';
     isScenario = false;  // 名称不包含"场景"
   }
   ```

2. **后端查询**（`test_cases.py` 第252行）：
   ```python
   # 只返回普通接口用例：名称不包含"场景"的用例
   query = query.filter(~TestCase.name.like('%场景%'))
   ```

3. **结果**：
   - 只显示名称不包含"场景"的测试用例
   - 例如："用户登录_测试用例"、"用户注册_测试用例"等
   - **不显示**名称包含"场景"的用例（如"场景_测试用例"）

**结论**："接口测试用例"标签页显示的是真正的接口测试用例（名称不包含"场景"）。

## 三个标签页的区别总结

| 标签页 | 查询条件 | 测试用例特征 | 生成方式 |
|--------|---------|------------|---------|
| **接口场景用例** | `case_type='pytest'`<br>`is_scenario=true`<br>名称包含"场景" | 名称包含"场景"<br>例如："场景_测试用例"<br>"其他_用户登录" | 1. 从场景用例集生成场景用例<br>2. 从接口场景用例标签页点击"生成接口用例"按钮 |
| **接口测试用例** | `case_type='pytest'`<br>`is_scenario=false`<br>名称不包含"场景" | 名称不包含"场景"<br>例如："用户登录_测试用例"<br>"用户注册_测试用例" | 1. 从场景用例集生成接口用例（`generate_type='interface'`）<br>2. 从接口列表手动生成<br>3. 创建接口测试任务时自动生成 |
| **性能测试用例** | `case_type='jmeter'` | JMeter格式的性能测试用例 | 从场景用例集生成性能用例 |

## 当前问题

### 按钮命名混淆

"接口场景用例"标签页中的"生成接口用例"按钮存在命名混淆：

- **按钮名称**："生成接口用例"（用户可能认为会生成接口测试用例）
- **实际行为**：生成的是接口场景用例（因为名称包含模块名，如果模块是"场景"，名称就是"场景_测试用例"）

### 建议

1. **重命名按钮**：将"生成接口用例"改为"生成场景接口用例"或"从场景生成接口用例"，更准确地描述其功能
2. **或者在"场景用例集"页面添加真正的"生成接口用例"按钮**：调用 `generate_type='interface'`，生成名称格式为 `{interface.name}_测试用例` 的接口测试用例

## 如何生成真正的接口测试用例

要生成真正的接口测试用例（显示在"接口测试用例"标签页），需要：

1. **从场景用例集页面生成**（如果后端支持 `generate_type='interface'`）：
   - 路径：场景用例集页面
   - 选择场景用例集
   - 点击"生成接口用例"按钮（需要后端支持 `generate_type='interface'`）
   - 生成的用例名称格式：`{interface.name}_测试用例`（不包含"场景"）

2. **从接口列表手动生成**：
   - 路径：接口列表页面
   - 选择接口
   - 点击"生成测试用例"按钮（如果存在）
   - 生成的用例名称格式：`{interface.name}_测试用例`

3. **创建接口测试任务时自动生成**：
   - 路径：测试任务页面
   - 创建接口测试任务
   - 不选择测试用例（留空）
   - 系统自动生成接口测试用例
