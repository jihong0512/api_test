# Backend 手动启动指南

## 📋 前提条件

1. 确保基础服务已启动（MySQL、Redis、Neo4j、MinIO）
2. 已配置 `.env` 文件（参考 `.env.example`）

## 🚀 启动步骤

### 方式1：使用 Conda（推荐）

```bash
# 进入backend目录
cd backend

# 创建conda环境（如果还没有）
conda create -n api_test python=3.12 -y

# 激活环境
conda activate api_test

# 安装依赖
pip install -r requirements.txt

# 启动服务（方式1：使用start_server.py脚本）
python start_server.py

# 或启动服务（方式2：直接使用uvicorn）
uvicorn main:app --reload --host 0.0.0.0 --port 8004
```

### 方式2：使用 Python venv

```bash
# 进入backend目录
cd backend

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
# Mac/Linux:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python start_server.py
# 或
uvicorn main:app --reload --host 0.0.0.0 --port 8004
```

## 🔧 启动命令说明

### 使用 start_server.py（推荐）

```bash
python start_server.py
```

**优点**：
- 自动读取环境变量 `SERVER_PORT`（默认8004）
- 支持热重载（代码修改自动重启）
- 友好的启动提示信息

**自定义端口**：
```bash
SERVER_PORT=8005 python start_server.py
```

### 使用 uvicorn 直接启动

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8004
```

**参数说明**：
- `main:app` - 从main.py导入app对象
- `--reload` - 开发模式，代码修改自动重启
- `--host 0.0.0.0` - 绑定所有网络接口
- `--port 8004` - 服务端口（默认8004）

## ⚠️ 注意事项

### 1. 端口配置

- **默认端口**: 8004
- 如果端口被占用，可以修改：
  ```bash
  # 方式1：环境变量
  SERVER_PORT=8005 python start_server.py
  
  # 方式2：uvicorn参数
  uvicorn main:app --reload --port 8005
  ```

### 2. 环境变量配置

确保 `.env` 文件已配置，特别是：
- `DEEPSEEK_API_KEY` - DeepSeek API密钥
- `QWEN_API_KEY` - Qwen API密钥
- 数据库连接配置（MySQL、Redis等）

### 3. 基础服务检查

启动前确保以下服务已运行：
- MySQL (端口3309，本地开发模式)
- Redis (端口6382，本地开发模式)
- Neo4j (端口7687)
- MinIO (端口9005，本地开发模式)

检查命令：
```bash
# 检查MySQL
docker-compose -f ../docker-compose.dev.yml ps mysql

# 检查Redis
docker-compose -f ../docker-compose.dev.yml ps redis

# 检查所有服务
docker-compose -f ../docker-compose.dev.yml ps
```

## 🔍 常见问题

### 1. 端口被占用

```bash
# 检查端口占用
lsof -i :8004

# 使用其他端口
SERVER_PORT=8005 python start_server.py
```

### 2. 模块导入错误

确保在 `backend` 目录下运行：
```bash
cd backend
python start_server.py
```

### 3. 数据库连接失败

检查 `.env` 文件中的数据库配置：
- `MYSQL_HOST=localhost`
- `MYSQL_PORT=3309`（本地开发模式）
- 确保MySQL容器正在运行

### 4. 依赖安装失败

```bash
# 升级pip
pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```
