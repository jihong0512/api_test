# 彻底解决文档解析卡住问题

## 问题诊断

如果文档一直显示"解析中"，可能的原因：
1. Celery Worker未运行或崩溃
2. 解析任务执行失败但未更新状态
3. 任务超时但未正确处理
4. 数据库状态未正确更新

## 解决方案

### 1. 检查文档实际状态

```bash
cd apitest/backend
python3 fix_stuck_documents.py
```

### 2. 重置卡住的文档（超过30分钟）

```bash
cd apitest/backend
python3 fix_stuck_documents.py --reset-all
```

### 3. 检查特定文档

```bash
cd apitest/backend
python3 fix_stuck_documents.py --document-id <文档ID>
```

### 4. 检查Celery Worker

```bash
# 检查Worker是否运行
ps aux | grep celery

# 查看日志
tail -f logs/celery_worker.log
```

### 5. 重启Celery Worker

```bash
# 停止Worker
pkill -f "celery.*worker"

# 启动Worker
cd apitest/backend
celery -A app.celery_app worker --loglevel=info --concurrency=4 > ../logs/celery_worker.log 2>&1 &
echo $! > ../logs/celery_worker.pid
```

### 6. 通过API重新解析文档

如果文档状态异常，可以通过API重新解析：

```bash
# 获取文档ID（从数据库或前端）
DOCUMENT_ID=<文档ID>

# 调用重新解析接口
curl -X POST "http://localhost:8009/api/documents/${DOCUMENT_ID}/retry"
```

## 已修复的问题

1. ✅ **重新解析接口增强**：现在会自动撤销旧任务
2. ✅ **错误处理改进**：确保异常时正确更新文档状态
3. ✅ **任务超时检测**：自动检测并修复卡住的任务
4. ✅ **修复工具**：提供命令行工具自动修复

## 预防措施

1. **定期检查**：定期运行 `fix_stuck_documents.py` 检查卡住的文档
2. **监控日志**：关注 `logs/celery_worker.log` 中的错误
3. **任务超时**：任务默认30分钟超时，超时后会自动标记为失败

## 如果问题仍然存在

1. 检查文件是否存在且可读
2. 检查文件格式是否正确
3. 查看Celery Worker日志中的详细错误信息
4. 尝试手动重新上传文档
