## 智能断言概述

### 什么是智能断言？

智能断言是 apitest 系统的核心功能之一，它能够：

1. **自动生成断言规则**：根据接口定义自动推断需要验证的字段
2. **智能状态码判断**：根据HTTP方法自动推断期望的状态码
3. **JSON路径提取**：支持从嵌套JSON响应中提取字段值进行断言
4. **多种断言类型**：支持相等、包含、大于、小于等多种断言方式
5. **LLM增强**：使用大模型理解接口语义，生成更准确的断言规则

### 核心优势

```
传统方式：
❌ 手动编写每个断言
❌ 容易遗漏关键字段
❌ 状态码判断不准确
❌ 嵌套JSON提取困难

智能断言：
✅ 自动生成断言规则
✅ 覆盖关键字段
✅ 根据HTTP方法智能判断状态码
✅ 支持复杂JSON路径提取
✅ LLM理解业务语义，生成更准确的断言
```

## 断言生成机制

### 1. 默认断言：根据HTTP方法智能判断

系统会根据HTTP方法自动生成默认的状态码断言：

```python
# backend/app/services/test_case_generator.py
def _generate_assertions_for_requests(self, api_interface: Dict[str, Any], assertions: Optional[List[Dict[str, Any]]]) -> str:
    """生成断言代码（requests库格式）"""
    code = ""
    method = api_interface.get("method", "").upper()
    
    # 默认断言：状态码
    if method == "GET":
        # GET请求通常返回200
        code += "    assert response.status_code == 200, f\"Expected status code 200, got {response.status_code}. Response: {response.text}\"\n"
    elif method == "POST":
        # POST请求创建资源，通常返回200或201
        code += "    assert response.status_code in [200, 201], f\"Expected status code 200 or 201, got {response.status_code}. Response: {response.text}\"\n"
    elif method in ["PUT", "PATCH"]:
        # PUT/PATCH更新资源，通常返回200
        code += "    assert response.status_code == 200, f\"Expected status code 200, got {response.status_code}. Response: {response.text}\"\n"
    elif method == "DELETE":
        # DELETE删除资源，通常返回200或204
        code += "    assert response.status_code in [200, 204], f\"Expected status code 200 or 204, got {response.status_code}. Response: {response.text}\"\n"
    else:
        # 其他方法，只要不是4xx/5xx错误即可
        code += "    assert response.status_code < 400, f\"Expected status code < 400, got {response.status_code}. Response: {response.text}\"\n"
```

**为什么这样设计？**

- **GET**：查询操作，成功返回200
- **POST**：创建操作，成功返回201（Created）或200
- **PUT/PATCH**：更新操作，成功返回200
- **DELETE**：删除操作，成功返回200或204（No Content）

### 3. 从接口定义自动生成断言

系统可以根据接口的响应Schema自动生成断言规则：

```python
def generate_assertions_from_schema(api_interface: Dict[str, Any]) -> List[Dict[str, Any]]:
    """
    根据接口定义自动生成断言规则
    """
    assertions = []
    response_schema = api_interface.get("response_schema", {})
    
    # 1. 状态码断言（根据HTTP方法）
    method = api_interface.get("method", "GET").upper()
    if method == "GET":
        expected_status = 200
    elif method == "POST":
        expected_status = 201
    elif method == "DELETE":
        expected_status = 204
    else:
        expected_status = 200
    
    assertions.append({
        "type": "status_code",
        "field": "status_code",
        "expected": expected_status
    })
    
    # 2. 响应体结构断言
    if response_schema and "properties" in response_schema:
        # 检查必需字段
        required_fields = response_schema.get("required", [])
        for field in required_fields:
            assertions.append({
                "type": "json_body",
                "field": f"body.{field}",
                "expected": "not_null",
                "description": f"字段 {field} 必须存在"
            })
        
        # 检查关键字段的类型
        properties = response_schema.get("properties", {})
        for field, field_schema in properties.items():
            field_type = field_schema.get("type")
            if field_type:
                assertions.append({
                    "type": "json_type",
                    "field": f"body.{field}",
                    "expected": field_type,
                    "description": f"字段 {field} 的类型应该是 {field_type}"
                })
    
    # 3. 业务逻辑断言（根据字段名推断）
    # 例如：如果响应中有 "code" 字段，通常表示业务状态码
    if "code" in str(response_schema):
        assertions.append({
            "type": "equal",
            "field": "body.code",
            "expected": 0,
            "description": "业务状态码应该为0（成功）"
        })
    
    # 4. 认证相关断言
    # 如果接口需要认证，通常响应中会有token或userId
    if api_interface.get("headers", {}).get("Authorization"):
        assertions.append({
            "type": "json_contains",
            "field": "body",
            "expected": "token",
            "description": "登录接口应该返回token"
        })
    
    return assertions
```

## 断言执行机制

### 1. 断言执行流程

当测试用例执行时，系统会按照以下流程执行断言：

```python
# backend/app/services/test_executor.py
def _execute_assertions(
    self,
    test_case: TestCase,
    status_code: int,
    response_data: Any
) -> List[Dict[str, Any]]:
    """执行断言"""
    assertions = []
    
    # 步骤1：执行默认断言（状态码）
    method = test_case.api_interface.method.upper()
    expected_status = 200
    if method == "POST":
        expected_status = 201
    elif method == "DELETE":
        expected_status = 204
    
    assertions.append({
        "type": "status_code",
        "expected": expected_status,
        "actual": status_code,
        "passed": status_code == expected_status
    })
    
    # 步骤2：执行自定义断言
    if test_case.assertions:
        custom_assertions = json.loads(test_case.assertions)
        for assertion in custom_assertions:
            assert_type = assertion.get("type", "equal")
            field = assertion.get("field", "")
            expected = assertion.get("expected", "")
            
            # 从响应中提取字段值
            actual = self._extract_field_value(response_data, field)
            
            # 根据断言类型判断是否通过
            passed = False
            if assert_type == "equal":
                passed = actual == expected
            elif assert_type == "not_equal":
                passed = actual != expected
            elif assert_type == "contains":
                passed = expected in str(actual)
            elif assert_type == "greater_than":
                passed = actual > expected
            elif assert_type == "less_than":
                passed = actual < expected
            
            assertions.append({
                "type": assert_type,
                "field": field,
                "expected": expected,
                "actual": actual,
                "passed": passed
            })
    
    return assertions
```

### 2. 断言结果结构

每个断言执行后都会返回详细的结果：

```python
assertion_result = {
```

## 字段提取机制

### 1. JSON路径提取

系统支持从嵌套的JSON响应中提取字段值，使用点号路径（dot notation）：

```python
# backend/app/services/test_executor.py
def _extract_field_value(self, data: Any, field_path: str) -> Any:
    """从响应中提取字段值"""
    if not field_path:
        return None
    
    # 处理字段路径前缀（body. 或 json.）
    if field_path.startswith("body."):
        field_path = field_path.replace("body.", "")
    elif field_path.startswith("json."):
        field_path = field_path.replace("json.", "")
    
    # 按点号分割路径
    parts = field_path.split(".")
    current = data
    
    # 逐层提取
    for part in parts:
        if isinstance(current, dict):
            current = current.get(part)
        elif isinstance(current, list) and part.isdigit():
            try:
                current = current[int(part)]
            except:
                return None
        else:
            return None
        
        if current is None:
            return None
    
    return current
```

### 2. 提取示例

**示例1：简单字段提取**

```python
# 响应数据
response_data = {
```

## LLM增强断言生成

### 1. LLM生成断言规则

系统可以使用LLM根据接口定义自动生成更智能的断言规则：

```python
# backend/app/services/llm_service.py
async def generate_test_case(
    self,
    api_info: Dict[str, Any],
    context: Optional[str] = None
) -> Dict[str, Any]:
    """生成测试用例（包含断言规则）"""
    prompt = f"""
基于以下API接口信息，生成测试用例：

接口信息：
{json.dumps(api_info, ensure_ascii=False, indent=2)}

{"上下文信息：" + context if context else ""}

请生成以下格式的测试用例：
{{
    "name": "测试用例名称",
    "description": "测试用例描述",
    "test_data": {{
        "params": {{}},
        "headers": {{}},
        "body": {{}}
    }},
    "assertions": [
        {{"type": "status_code", "expected": 200}},
        {{"type": "response_time", "expected": 1000}},
        {{"type": "contains", "field": "data", "value": ""}}
    ]

请输出JSON格式：
"""
    result = await self.chat(prompt, temperature=0.5)
    return json.loads(result)
```

## 完整代码示例

### 示例1：完整的断言生成和执行流程

```python
from typing import Dict, Any, List, Optional
import json
import requests

class IntelligentAssertionSystem:
    """智能断言系统"""
    
    def __init__(self):
        self.llm_service = None  # LLM服务（可选）
    
    def generate_assertions(
        self,
        api_interface: Dict[str, Any],
        use_llm: bool = False
    ) -> List[Dict[str, Any]]:
        """
        生成断言规则
        
        Args:
            api_interface: 接口定义
            use_llm: 是否使用LLM增强
        
        Returns:
            断言规则列表
        """
        assertions = []
        
        # 1. 默认状态码断言
        method = api_interface.get("method", "GET").upper()
        expected_status = self._get_expected_status(method)
        assertions.append({
            "type": "status_code",
            "field": "status_code",
            "expected": expected_status
        })
        
        # 2. 从响应Schema生成断言
        response_schema = api_interface.get("response_schema", {})
        if response_schema:
            schema_assertions = self._generate_from_schema(response_schema)
            assertions.extend(schema_assertions)
        
        # 3. LLM增强（可选）
        if use_llm and self.llm_service:
            llm_assertions = self._generate_with_llm(api_interface)
            assertions.extend(llm_assertions)
        
        return assertions
    
    def _get_expected_status(self, method: str) -> int:
        """根据HTTP方法获取期望状态码"""
        status_map = {
            "GET": 200,
            "POST": 201,
            "PUT": 200,
            "PATCH": 200,
            "DELETE": 204
        }
        return status_map.get(method, 200)
    
    def _generate_from_schema(
        self,
        response_schema: Dict[str, Any]
    ) -> List[Dict[str, Any]]:
        """从响应Schema生成断言"""
        assertions = []
        
        # 检查必需字段
        required_fields = response_schema.get("required", [])
        for field in required_fields:
            assertions.append({
                "type": "json_body",
                "field": f"body.{field}",
                "expected": "not_null"
            })
        
        # 检查关键业务字段
        properties = response_schema.get("properties", {})
        for field, field_schema in properties.items():
            # 业务状态码字段
            if field == "code":
                assertions.append({
                    "type": "equal",
                    "field": "body.code",
                    "expected": 0,
                    "description": "业务状态码应该为0（成功）"
                })
            
            # token字段
            if field == "token" or "token" in field.lower():
                assertions.append({
                    "type": "json_body",
                    "field": f"body.{field}",
                    "expected": "not_null",
                    "description": f"字段 {field} 必须存在且不为空"
                })
        
        return assertions
    
    def _generate_with_llm(
        self,
        api_interface: Dict[str, Any]
    ) -> List[Dict[str, Any]]:
        """使用LLM生成断言规则"""
        prompt = f"""
根据以下接口定义，生成智能断言规则：

接口信息：
{json.dumps(api_interface, ensure_ascii=False, indent=2)}

请分析接口的业务含义，生成合适的断言规则。
重点关注：
1. 业务状态码（code字段）
2. 关键数据字段（token、id、userId等）
3. 数据类型验证
4. 业务逻辑验证

请返回JSON格式的断言规则列表：
[
    {{"type": "equal", "field": "body.code", "expected": 0}},
    {{"type": "json_body", "field": "body.data.token", "expected": "not_null"}}
]
"""
        result = self.llm_service.chat(prompt, temperature=0.3)
        return json.loads(result)
    
    def execute_assertions(
        self,
        response: requests.Response,
        assertions: List[Dict[str, Any]]
    ) -> List[Dict[str, Any]]:
        """
        执行断言
        
        Args:
            response: HTTP响应对象
            assertions: 断言规则列表
        
        Returns:
            断言执行结果列表
        """
        results = []
        
        # 解析响应数据
        try:
            response_data = response.json()
        except:
            response_data = {"text": response.text}
        
        # 执行每个断言
        for assertion in assertions:
            assert_type = assertion.get("type")
            field = assertion.get("field", "")
            expected = assertion.get("expected")
            
            # 提取实际值
            if field == "status_code":
                actual = response.status_code
            else:
                actual = self._extract_field_value(response_data, field)
            
            # 判断是否通过
            passed = self._evaluate_assertion(assert_type, actual, expected)
            
            results.append({
                "type": assert_type,
                "field": field,
                "expected": expected,
                "actual": actual,
                "passed": passed
            })
        
        return results
    
    def _extract_field_value(
        self,
        data: Any,
        field_path: str
    ) -> Any:
        """从响应中提取字段值"""
        if not field_path:
            return None
        
        # 处理前缀
        if field_path.startswith("body."):
            field_path = field_path.replace("body.", "")
        elif field_path.startswith("json."):
            field_path = field_path.replace("json.", "")
        
        # 提取字段
        parts = field_path.split(".")
        current = data
        
        for part in parts:
            if isinstance(current, dict):
                current = current.get(part)
            elif isinstance(current, list) and part.isdigit():
                try:
                    current = current[int(part)]
                except:
                    return None
            else:
                return None
            
            if current is None:
                return None
        
        return current
    
    def _evaluate_assertion(
        self,
        assert_type: str,
        actual: Any,
        expected: Any
    ) -> bool:
        """评估断言是否通过"""
        if assert_type == "equal":
            return actual == expected
        elif assert_type == "not_equal":
            return actual != expected
        elif assert_type == "contains":
            return expected in str(actual)
        elif assert_type == "greater_than":
            return actual > expected
        elif assert_type == "less_than":
            return actual < expected
        elif assert_type == "json_body":
            if expected == "not_null":
                return actual is not None
            return actual == expected
        elif assert_type == "status_code":
            return actual == expected
        else:
            return False


# ===== 使用示例 =====
```

## 最佳实践

### ✅ 推荐做法

1. **使用默认断言**：让系统根据HTTP方法自动生成状态码断言
2. **覆盖关键字段**：为重要的业务字段添加断言（如code、token、id等）
3. **使用JSON路径**：对于嵌套JSON，使用点号路径提取字段
4. **启用LLM增强**：对于复杂接口，使用LLM生成更智能的断言规则
5. **添加描述信息**：为每个断言添加清晰的描述，便于调试

### ❌ 避免的做法

1. ❌ 只断言状态码，不验证响应内容
2. ❌ 使用硬编码的字段路径，不处理嵌套结构
3. ❌ 忽略业务逻辑，只做简单的相等判断
4. ❌ 不处理异常情况（如响应不是JSON格式）
