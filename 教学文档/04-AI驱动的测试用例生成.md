### 核心目标
```
从API接口定义 → 自动生成 → 可执行的高质量测试用例
```

### 为什么需要AI驱动？

**传统方式的问题**：
- ❌ 手动编写用例耗时（每个接口多个用例）
- ❌ 用例覆盖率不全（遗漏边界情况）
- ❌ 维护成本高（接口变更需要手动更新）
- ❌ 测试数据难以构造（特别是复杂业务场景）

**AI驱动的优势**：
- ✅ 自动生成，效率高（一键生成多个用例）
- ✅ 覆盖率高（AI能考虑边界和异常情况）
- ✅ 自动维护（接口变化自动重新生成）
- ✅ 智能数据生成（AI理解业务逻辑构造合理数据）

## Few-shot学习与RAG检索

### Few-shot学习：让LLM学会"怎么写"

**核心思想**：
```
与其告诉LLM规则，不如给它看"好作文"
```

**比喻**：
```
教小孩写作文的方式：

方式1（规则式）：
"你要写：开头段落+中间论述+结尾总结"

方式2（Few-shot式）：
"看这3篇好作文，然后你也这样写"

显然方式2效果更好！LLM也是如此。
```

#### 完整的Few-shot + RAG实现代码

```python
import httpx
import json
from typing import List, Dict, Any
from sentence_transformers import SentenceTransformer
import numpy as np

class TestCaseGenerator:
    """使用Few-shot + RAG生成测试用例"""
    
    def __init__(self, deepseek_api_key: str):
        self.api_key = deepseek_api_key
        self.client = httpx.AsyncClient()
        
        # 初始化embedding模型（用于RAG检索）
        self.embedding_model = SentenceTransformer('paraphrase-MiniLM-L6-v2')
        
        # 存储历史用例（实际应该从数据库加载）
        self.historical_cases = []
    
    def _build_few_shot_prompt(self) -> str:
        """构造Few-shot提示词"""
        return """
【参考示例1：登录接口测试】
接口: POST /api/auth/login
请求: {username: "test", password: "pwd123"}
响应: {code: 0, data: {token: "xxx", userId: 123}}

生成代码:
def test_login():
    resp = requests.post(f"{BASE_URL}/api/auth/login",
                        json={"username": "test", "password": "pwd123"})
    assert resp.status_code == 200
    assert resp.json()["code"] == 0
    assert "token" in resp.json()["data"]

【参考示例2：需要认证的接口】
接口: GET /api/users/{userId}
需要: Authorization: Bearer {token}
响应: {code: 0, data: {id, name, email}}

生成代码:
@pytest.fixture
def login_token():
    resp = requests.post(f"{BASE_URL}/api/auth/login",
                        json={"username": "test", "password": "pwd123"})
    return resp.json()["data"]["token"]

def test_get_user(login_token):
    resp = requests.get(f"{BASE_URL}/api/users/123",
                       headers={"Authorization": f"Bearer {login_token}"})
    assert resp.status_code == 200
    assert resp.json()["code"] == 0
    assert "name" in resp.json()["data"]
"""
    
    async def retrieve_similar_cases(self, 
                                    interface_info: Dict[str, Any],
                                    top_k: int = 3) -> List[str]:
        """
        从历史用例中检索相似的（RAG）
        """
        # 1️⃣ 将当前接口转换为向量
        query_text = f"{interface_info['name']} {interface_info.get('description', '')}"
        query_vector = self.embedding_model.encode(query_text)
        
        # 2️⃣ 计算与历史用例的相似度
        similarities = []
        for i, case in enumerate(self.historical_cases):
            case_text = f"{case['name']} {case.get('description', '')}"
            case_vector = self.embedding_model.encode(case_text)
            
            # 使用余弦相似度
            similarity = np.dot(query_vector, case_vector) / (
                np.linalg.norm(query_vector) * np.linalg.norm(case_vector)
            )
            similarities.append((similarity, i, case))
        
        # 3️⃣ 返回最相似的top_k个用例
        similarities.sort(reverse=True, key=lambda x: x[0])
        
        similar_code_examples = []
        for _, _, case in similarities[:top_k]:
            similar_code_examples.append(case.get("code", ""))
        
        return similar_code_examples
    
    async def generate_test_case(self, interface_info: Dict[str, Any]) -> str:
        """
        生成测试用例
        完整流程：Few-shot + RAG + LLM
        """
        # 1️⃣ Few-shot提示词
        few_shot = self._build_few_shot_prompt()
        
        # 2️⃣ RAG检索相似用例
        similar_cases = await self.retrieve_similar_cases(interface_info)
        similar_context = "\n\n".join(similar_cases) if similar_cases else ""
        
        # 3️⃣ 构造完整提示词
        system_prompt = """你是专业的API测试工程师，生成高质量的Pytest测试用例。"""
        
        user_prompt = f"""
【Few-shot参考示例】
{few_shot}

【项目相似用例（参考）】
{similar_context}

【要生成用例的接口】
- 名称: {interface_info['name']}
- 方法: {interface_info['method']}
- URL: {interface_info['url']}
- 描述: {interface_info.get('description', '')}
- 请求参数: {interface_info.get('request_body', {})}
- 响应格式: {interface_info.get('response_body', {})}
- 依赖: {interface_info.get('dependencies', [])}

请根据以上参考示例和项目风格，生成该接口的测试用例代码。
只返回Python代码，不需要其他说明。
"""
        
        # 4️⃣ 调用LLM生成
        response = await self.client.post(
            "https://api.deepseek.com/v1/chat/completions",
            headers={
                "Authorization": f"Bearer {self.api_key}",
                "Content-Type": "application/json"
            },
            json={
                "model": "deepseek-chat",
                "messages": [
                    {"role": "system", "content": system_prompt},
                    {"role": "user", "content": user_prompt}
                ],
                "temperature": 0.3,  # 相对确定
                "max_tokens": 1000
            }
        )
        
        result = response.json()
        test_code = result["choices"][0]["message"]["content"]
        
        return test_code
    
    def add_historical_case(self, name: str, description: str, code: str):
        """添加历史用例，用于RAG检索"""
        self.historical_cases.append({
            "name": name,
            "description": description,
            "code": code
        })
```

### RAG检索：让LLM利用"历史知识"

**核心思想**：
```
不要让LLM从零开始生成
而是从历史中找到相似的例子，作为参考
```

#### RAG的完整流程

**步骤1：构建知识库**
```
项目历史上所有生成过的测试用例
  ↓
提取关键信息（接口名、业务类型、参数特征等）
  ↓
生成向量表示（embedding）
  ↓
存入向量数据库（ChromaDB/Milvus）
```

**步骤2：检索相似用例**
```
当要为"登录相关的新接口"生成用例时：
  ↓
将这个新接口转换为向量
  ↓
在向量库中查找"最相似"的历史用例
  ↓
返回相似度最高的3-5个例子
```

**步骤3：LLM参考历史生成新用例**
```
LLM获得：
  1. 新接口的详细信息
  2. 相似接口的历史用例
  3. Few-shot范例
  
基于这些上下文，生成新的测试用例
```

```python
async def retrieve_similar_cases(interface_info, project_id, top_k=3):
    """
    从向量库中检索相似的测试用例
    """
    # 1. 提取当前接口的特征文本
    query_text = f"""
    接口名: {interface_info['name']}
    方法: {interface_info['method']}
    描述: {interface_info['description']}
    参数: {json.dumps(interface_info['parameters'])}
    """
    
    # 2. 转换为向量
    query_vector = await embedding_model.encode(query_text)
    
    # 3. 在向量库中搜索
    results = await vector_db.search(
        query_vector,
        collection=f"project_{project_id}_test_cases",
        limit=top_k,
        threshold=0.7  # 相似度至少70%
    )
    
    # 4. 从MySQL获取完整的用例代码
    similar_cases = []
    for result in results:
        case = await db.query(TestCase).filter(
            TestCase.id == result['metadata']['case_id']
        ).first()
        similar_cases.append({
            "interface_name": case.interface_name,
            "code": case.code,
            "similarity": result['score']
        })
    
    return similar_cases
```

```
输入：新接口信息
  ↓
```

### HttpRunner风格（推荐）

这是一种结构化的测试用例格式，更易读且易于维护

```python
import pytest
import requests
from typing import Dict, Any, Optional, List

class TestCase:
    """基础测试用例类"""
    
    def __init__(self, 
                 name: str,
                 method: str,
                 url: str,
                 data: Optional[Dict[str, Any]] = None,
                 headers: Optional[Dict[str, str]] = None,
                 assertions: Optional[List[Dict[str, Any]]] = None,
                 setup_steps: Optional[List[Dict[str, Any]]] = None
    ):
        self.name = name
        self.method = method
        self.url = url
        self.data = data or {}
        self.headers = headers or {}
        self.assertions = assertions or []
        self.setup_steps = setup_steps or []
        self.session = requests.Session()
        self.extracted_data = {}
    
    async def run(self, base_url: str) -> Dict[str, Any]:
        """执行测试用例"""
        # 执行前置步骤
        for step in self.setup_steps:
            await self._execute_step(base_url, step)
        
        # 替换参数中的变量
        url = self._replace_variables(base_url + self.url)
        data = self._replace_variables_in_dict(self.data)
        headers = self._replace_variables_in_dict(self.headers)
        
        # 发送请求
        full_url = url if url.startswith("http") else base_url + url
        response = self.session.request(
            self.method,
            full_url,
            json=data if self.method in ["POST", "PUT", "PATCH"] else None,
            params=data if self.method == "GET" else None,
            headers=headers
        )
        
        # 执行断言
        result = await self._execute_assertions(response)
        
        # 提取响应数据
        await self._extract_data(response)
        
        return result
    
    def _replace_variables(self, text: str) -> str:
        """替换文本中的变量"""
        for key, value in self.extracted_data.items():
            text = text.replace(f"${{{key}}}", str(value))
        return text
    
    def _replace_variables_in_dict(self, d: Dict) -> Dict:
        """递归替换字典中的变量"""
        result = {}
        for key, value in d.items():
            if isinstance(value, str):
                result[key] = self._replace_variables(value)
            elif isinstance(value, dict):
                result[key] = self._replace_variables_in_dict(value)
            else:
                result[key] = value
        return result
    
    async def _execute_assertions(self, response) -> bool:
        """执行断言"""
        for assertion in self.assertions:
            assertion_type = assertion.get("type")  # status_code, json_body, etc
            
            if assertion_type == "status_code":
                assert response.status_code == assertion["expected"], \
                    f"预期状态码 {assertion['expected']}, 实际 {response.status_code}"
            
            elif assertion_type == "json_body":
                path = assertion.get("path")  # "code", "data.id"
                expected = assertion.get("expected")
                actual = self._get_json_value(response.json(), path)
                assert actual == expected, f"{path} 预期值 {expected}, 实际值 {actual}"
            
            elif assertion_type == "json_contains":
                path = assertion.get("path")
                expected = assertion.get("expected")
                actual = self._get_json_value(response.json(), path)
                assert expected in actual, f"{path} 应包含 {expected}"
        
        return True
    
    def _get_json_value(self, obj: Dict, path: str):
        """根据路径获取JSON中的值"""
        keys = path.split(".")
        current = obj
        for key in keys:
            if isinstance(current, dict):
                current = current.get(key)
            else:
                return None
        return current
    
    async def _extract_data(self, response):
        """从响应中提取数据"""
        try:
            data = response.json()
            for assertion in self.assertions:
                if "extract" in assertion:
                    path = assertion["extract"]
                    key = assertion.get("extract_as")
                    value = self._get_json_value(data, path)
                    if value is not None:
                        self.extracted_data[key] = value
        except:
            pass
```

```python
class TestCaseGenerator:
    """测试用例生成器"""
    
    def __init__(self):
        self.data_generator = SmartTestDataGenerator()
        self.llm_service = LLMService()
    
    async def generate(self, 
                      api_interface: Dict[str, Any],
                      dependencies: Optional[Dict[str, Any]] = None,
                      use_llm: bool = True) -> str:
        """
        生成测试用例
        """
        # 步骤1：分析接口信息
        api_info = self._analyze_interface(api_interface)
        
        # 步骤2：生成测试数据
        test_data = await self._generate_test_data(api_info)
        
        # 步骤3：处理依赖关系
        extracted_data = {}
        setup_steps = []
        if dependencies:
            extracted_data, setup_steps = await self._handle_dependencies(
                dependencies
            )

        if use_llm:
            code = await self._generate_with_llm(
                api_info,

            )
        else:

                api_info,
                test_data,
                extracted_data,
                setup_steps
            )

    def _analyze_interface(self, api_interface: Dict[str, Any]) -> Dict:
        """分析接口信息"""
        return {
            "name": api_interface.get("name"),
            "method": api_interface.get("method", "GET").upper(),
            "url": api_interface.get("url"),
            "description": api_interface.get("description"),
            "params": json.loads(api_interface.get("params", "{}")),
            "headers": json.loads(api_interface.get("headers", "{}")),
            "request_body": json.loads(api_interface.get("request_body", "{}")),
            "response_schema": json.loads(api_interface.get("response_schema", "{}"))
        }
    
    async def _generate_test_data(self, api_info: Dict) -> Dict[str, Any]:
        """
        根据参数类型生成测试数据
        """
        test_data = {}
        
        # 生成查询参数
        for param_name, param_schema in api_info["params"].items():
            test_data[param_name] = self._generate_value_for_type(param_schema)
        
        # 生成请求体
        if api_info["request_body"]:
            request_body = api_info["request_body"]
            if isinstance(request_body, dict):
                for field_name, field_schema in request_body.get("properties", {}).items():
                    test_data[field_name] = self._generate_value_for_type(field_schema)
        
        return test_data
    
    def _generate_value_for_type(self, schema: Dict) -> Any:
        """根据schema生成测试值"""
        field_type = schema.get("type", "string")
        
        if field_type == "string":
            description = schema.get("description", "").lower()
            
            # 智能识别字段含义
            if "email" in description:
                return "test@example.com"
            elif "password" in description:
                return "TestPass123!"
            elif "username" in description:
                return "testuser"
            elif "name" in description:
                return "Test User"
            elif "phone" in description:
                return "13800000000"
            elif "url" in description:
                return "http://example.com"
            else:
                return "test_value"
        
        elif field_type == "integer":
            return 123
        
        elif field_type == "number":
            return 123.45
        
        elif field_type == "boolean":
            return True
        
        elif field_type == "array":
            items_schema = schema.get("items", {"type": "string"})
            return [self._generate_value_for_type(items_schema)]
        
        elif field_type == "object":
            obj = {}
            for prop_name, prop_schema in schema.get("properties", {}).items():
                obj[prop_name] = self._generate_value_for_type(prop_schema)
            return obj
        
        else:
            return None
    
    async def _generate_with_llm(self,
                                api_info: Dict,
                                test_data: Dict,
                                extracted_data: Dict,
                                setup_steps: List) -> str:
        """使用LLM生成测试代码"""

        {json.dumps(test_data, ensure_ascii=False, indent=2)}

        {json.dumps(extracted_data, ensure_ascii=False, indent=2)}
        
        前置步骤：
        {json.dumps(setup_steps, ensure_ascii=False, indent=2)}
        
        请生成以下内容的Pytest代码：
        1. 导入必要的库
        2. 定义测试类
        3. 设置测试数据
        4. 发送HTTP请求
        5. 验证响应状态和内容
        6. 提取响应数据用于后续步骤
        
        代码应该：
        - 使用requests库发送HTTP请求
        - 包含合适的断言

    def _generate_with_template(self,

        # 替换占位符

            method_lower=api_info["method"].lower(),

            extracted_data_init=self._generate_extracted_data_init(extracted_data)
```

## 断言规则设计

### 断言类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `status_code` | HTTP状态码 | `assert response.status_code == 200` |
| `json_body` | JSON响应体的值 | `assert data["code"] == 0` |
| `json_contains` | JSON包含某个值 | `assert "token" in data` |
| `response_time` | 响应时间 | `assert response.elapsed.total_seconds() < 1` |
| `header_exists` | 响应头存在 | `assert "Set-Cookie" in response.headers` |

```python
def generate_assertions(api_interface: Dict, response_schema: Dict) -> List[Dict]:
    """
    根据接口定义自动生成断言规则
    """
    assertions = []
    
    # 1. 状态码断言
    status_code = api_interface.get("status_code", 200)
    assertions.append({
        "type": "status_code",
        "expected": status_code
    })
    
    # 2. 响应头断言（如果定义了）
    if "response_headers" in api_interface:
        for header, value in api_interface["response_headers"].items():
            assertions.append({
                "type": "header_exists",
                "header": header
            })
    
    # 3. 响应体断言
    if response_schema and "properties" in response_schema:
        for field, field_schema in response_schema["properties"].items():
            if field_schema.get("required"):
                assertions.append({
                    "type": "json_body",
                    "path": field,
                    "expected": "not_null"
                })
            
            # 如果字段有type，添加类型校验
            if "type" in field_schema:
                assertions.append({
                    "type": "json_type",
                    "path": field,
                    "expected": field_schema["type"]
                })
    
    # 4. 提取数据用于后续步骤
    important_fields = ["id", "token", "userId", "orderId"]
    for field in important_fields:
        if field in response_schema.get("properties", {}):
            assertions.append({
                "type": "extract",
                "path": f"data.{field}",
                "extract_as": field
            })
    
    return assertions
```

## 最佳实践总结

### ✅ 推荐做法

1. **使用HttpRunner格式** - 结构清晰，易于维护
2. **智能生成测试数据** - 识别字段含义，生成合理的值
3. **自动生成断言** - 根据响应schema自动添加验证
4. **处理依赖关系** - 确保前置条件满足
5. **提取响应数据** - 供后续步骤使用
6. **使用LLM增强** - 生成更加完整和智能的用例
7. **定期校验覆盖率** - 发现和补全缺失用例
8. **验证用例正确性** - 确保生成的用例可执行

### ⚠️ 常见陷阱

1. ❌ 生成的测试数据不合理（如密码只是"test"）
2. ❌ 遗漏必要的断言
3. ❌ 未正确处理前置依赖
4. ❌ 忽视错误响应的测试
5. ❌ 未提取关键响应数据
6. ❌ 盲目相信自动生成，不进行覆盖率检查
7. ❌ 忽视用例的正确性验证
