## 💡 核心知识速记

### 系统做什么？
```
输入：各种格式的API文档
处理：解析 → 分析 → 理解
输出：可直接运行的测试代码
```

### 系统如何工作？
```
1️⃣ 文档解析（文档02）
   支持：Swagger、JSON、YAML、PDF、Word、Excel等
   输出：结构化接口信息

2️⃣ 依赖分析（文档03）
   识别：参数依赖、业务流程、认证关系
   输出：依赖关系和执行顺序

3️⃣ 用例生成（文档04）
   基于：接口定义、依赖关系、AI理解
   输出：Pytest格式的Python测试代码

4️⃣ 知识增强（文档05）
   技术：向量化、RAG、混合检索
   作用：提升LLM生成质量
```

### 为什么用AI？
```
❌ 传统方式：手写每个接口的用例，耗时且遗漏多
✅ AI方式：自动生成高覆盖率的用例，快速且准确
```

## ❓ 常见问题快速答案

**Q: 应该先看源代码还是先读文档？**
A: 先读文档理解概念，再看源代码理解实现。文档是地图，代码是细节。

**Q: 这些文档与官方文档有什么区别？**
A: 这些是**教学文档**，聚焦核心流程和重点概念。官方文档可能更全面但也更冗长。

**Q: 文档中的代码能直接运行吗？**
A: 文档中的代码是**简化示例**，展示思路和关键部分。完整代码在源文件中。

**Q: 学完这些文档就能开发吗？**
A: 能做很多事，但深入开发还需要看源代码和实践。这些文档是**起点**而非终点。

**Q: 如果遇到文档不清楚的地方怎么办？**
A: 1. 看QUICK_REFERENCE 2. 查看对应源代码 3. 参考相关的学习文档PDF。
