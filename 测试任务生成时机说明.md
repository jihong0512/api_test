# 测试任务生成时机说明

## 重要说明

**测试任务不是"生成的"，而是"用户手动创建的"**。

测试任务通过前端UI由用户主动创建，系统根据用户的选择自动确定任务类型（接口场景测试任务 或 接口测试任务）。

---

## 创建路径

**前端页面**：`测试任务` 页面

**操作步骤**：
1. 用户进入"测试任务"页面
2. 点击"添加任务"按钮
3. 填写任务信息并选择用例/用例集
4. 系统根据选择自动确定任务类型

---

## 任务类型推断逻辑

**代码位置**：`backend/app/routers/test_tasks.py` 第110-114行

```python
# 推断执行类型：如果选择了用例集但未显式指定类型，强制视为场景执行
if suite_ids and (task.execution_task_type is None or task.execution_task_type == "interface"):
    task.execution_task_type = "scenario"  # ✅ 接口场景测试任务
elif not suite_ids and not task.execution_task_type:
    task.execution_task_type = "interface"  # ✅ 接口测试任务
```

### 规则说明

| 用户选择 | execution_task_type参数 | 系统推断结果 | 任务类型 |
|---------|------------------------|-------------|---------|
| **选择了用例集**（`test_case_suite_id` 或 `test_case_suite_ids`） | `None` 或 `"interface"` | **强制设为 `"scenario"`** | 接口场景测试任务 |
| **选择了用例集** | `"scenario"` | 保持 `"scenario"` | 接口场景测试任务 |
| **直接选择用例**（`test_case_ids`） | `None` | **设为 `"interface"`** | 接口测试任务 |
| **直接选择用例** | `"interface"` | 保持 `"interface"` | 接口测试任务 |

## 接口场景测试任务的创建时机

### 触发条件

**代码位置**：`backend/app/routers/test_tasks.py` 第84-101行、第135-138行

1. **用户在前端选择用例集**：
   - 提供 `test_case_suite_id` 或 `test_case_suite_ids` 参数
   - 系统从用例集中获取所有用例ID

2. **系统自动推断**（第111-112行）：
   ```python
   if suite_ids and (task.execution_task_type is None or task.execution_task_type == "interface"):
       task.execution_task_type = "scenario"  # ✅ 强制设为场景任务
   ```

3. **验证逻辑**（第135-138行）：
   ```python
   elif task.execution_task_type == "scenario":
       # 接口场景用例执行任务只能选择场景用例（从test_case_suites中选择）
       if not suite_ids:
           raise HTTPException(status_code=400, detail="接口场景用例执行任务必须选择测试用例组（test_case_suite_id或test_case_suite_ids）")
   ```

**结论**：
- ✅ 用户在前端选择场景用例集（`test_case_suite_id` 或 `test_case_suite_ids`）
- ✅ 系统自动将任务类型设为 `"scenario"`（接口场景测试任务）
- ✅ 系统从用例集中提取所有用例ID

## 接口测试任务的创建时机

### 触发条件

**代码位置**：`backend/app/routers/test_tasks.py` 第103-105行、第139-147行

1. **用户在前端直接选择用例**：
   - 提供 `test_case_ids` 参数（用例ID列表）
   - 不提供 `test_case_suite_id` 或 `test_case_suite_ids`

2. **系统自动推断**（第113-114行）：
   ```python
   elif not suite_ids and not task.execution_task_type:
       task.execution_task_type = "interface"  # ✅ 设为接口任务
   ```

3. **验证逻辑**（第139-147行）：
   ```python
   elif task.execution_task_type == "interface":
       # 接口测试用例执行任务只能选择pytest类型的用例
       valid_cases = db.query(TestCase).filter(
           TestCase.id.in_(test_case_ids),
           TestCase.project_id == project_id,
           TestCase.case_type == "pytest"
       ).all()
       if len(valid_cases) != len(test_case_ids):
           raise HTTPException(status_code=400, detail="接口测试用例执行任务只能选择接口测试用例（pytest类型）")
   ```

**结论**：
- ✅ 用户在前端直接选择测试用例（`test_case_ids`）
- ✅ 系统自动将任务类型设为 `"interface"`（接口测试任务）
- ✅ 系统验证所选用例必须是 `case_type='pytest'` 类型

## 完整创建流程

### 步骤1：用户在前端填写表单

**代码位置**：`frontend/src/pages/TestTasks.js` 第670-800行

用户在前端"新建测试任务"表单中填写：
- 任务名称
- 执行场景描述
- 任务类型（立即执行 或 定时任务）
- **执行任务类型**（可选，前端下拉选择）
- **测试用例** 或 **用例集**（二选一）
- 测试环境

### 步骤2：前端提交请求

**代码位置**：`frontend/src/pages/TestTasks.js` 第342-364行

```javascript
const handleSubmit = async () => {
  const values = await form.validateFields();
  const payload = {
    ...values,
  };
  
  await client.post(`/api/test-tasks/?project_id=${id}`, payload);
  // ...
}
```

### 步骤3：后端处理请求

**代码位置**：`backend/app/routers/test_tasks.py` 第49-239行

1. **获取用例ID列表**（第72-108行）
   - 如果提供了 `test_case_suite_ids` 或 `test_case_suite_id`：
     - 从用例集中提取所有用例ID
     - `suite_ids` 不为空
   - 如果提供了 `test_case_ids`：
     - 直接使用提供的用例ID
     - `suite_ids` 为空

2. **推断执行类型**（第110-114行）
   ```python
   if suite_ids and (task.execution_task_type is None or task.execution_task_type == "interface"):
       task.execution_task_type = "scenario"  # ✅ 接口场景测试任务
   elif not suite_ids and not task.execution_task_type:
       task.execution_task_type = "interface"  # ✅ 接口测试任务
   ```

3. **验证任务类型和用例类型的匹配**（第119-155行）
   - 接口场景测试任务：必须选择用例集
   - 接口测试任务：只能选择pytest类型的用例

4. **创建任务记录**（第190-209行）
   ```python
   db_task = TestTask(
       project_id=project_id,
       name=task.name,
       execution_task_type=task.execution_task_type,  # "scenario" 或 "interface"
       test_case_ids=json.dumps(sorted_case_ids),
       test_case_suite_id=test_case_suite_id,  # 场景任务会有这个字段
       environment_id=task.environment_id,
       status="pending"
   )
   ```

5. **如果立即执行，启动任务**（第212-216行）
   ```python
   if task.task_type == "immediate":
       task_scheduler.execute_task(db_task.id)
   ```

## 关键区别总结

| 特征 | 接口场景测试任务 | 接口测试任务 |
|------|----------------|------------|
| **用户选择** | 选择用例集（`test_case_suite_id` 或 `test_case_suite_ids`） | 直接选择用例（`test_case_ids`） |
| **系统推断** | `execution_task_type = "scenario"` | `execution_task_type = "interface"` |
| **用例来源** | 从用例集中提取所有用例ID | 用户直接提供的用例ID列表 |
| **用例类型要求** | 场景用例（从 `TestCaseSuite` 中选择） | pytest类型的用例（`case_type='pytest'`） |
| **执行方式** | 场景用例按顺序执行，可以传递数据（如token） | 每个用例独立执行 |
| **验证规则** | 必须选择用例集 | 只能选择pytest类型的用例 |

## 总结

1. **测试任务不是自动生成的**，而是用户通过前端UI手动创建的。

2. **接口场景测试任务**的创建时机：
   - 用户选择场景用例集（`test_case_suite_id` 或 `test_case_suite_ids`）
   - 系统自动将任务类型设为 `"scenario"`

3. **接口测试任务**的创建时机：
   - 用户直接选择测试用例（`test_case_ids`）
   - 系统自动将任务类型设为 `"interface"`

4. **系统会根据用户的选择自动推断任务类型**，用户也可以在前端显式指定任务类型。
