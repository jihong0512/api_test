## 完整流程

### 第一步：获取测试用例和API接口信息

**代码**（第2128-2139行）：
```python
# 获取测试用例记录
test_case = db.query(TestCase).filter(TestCase.id == test_case_id).first()
if not test_case:
    raise Exception(f"测试用例不存在: {test_case_id}")

# 获取API接口信息（优先从APIInterface表，如果不存在则从DocumentAPIInterface表）
api_interface = db.query(APIInterface).filter(APIInterface.id == api_interface_id).first()
if not api_interface:
    api_interface = db.query(DocumentAPIInterface).filter(DocumentAPIInterface.id == api_interface_id).first()
    if not api_interface:
        raise Exception(f"API接口不存在: {api_interface_id}")
```

**关键点**：
- ✅ 只从数据库读取接口定义信息（不读取业务数据）
- ✅ 从 `APIInterface` 表或 `DocumentAPIInterface` 表获取接口元数据

**关键点**：
- ✅ 只提取接口定义信息（方法、URL、参数、请求体等）

## 关键结论：是否关联数据库数据？

### ❌ 不关联业务数据库数据

**证据1**：代码中没有调用数据生成器
- 在 `generate_test_case_task` 中，**没有调用 `SmartTestDataGenerator.generate_test_data_for_api()`**
- 没有传递 `connection_id` 参数
- 没有连接业务数据库

**证据2**：生成器调用时没有传递测试数据
```python
test_code = generator.generate_test_case(
    api_interface=interface_info,  # 只传入接口信息
    project_id=project_id
    # ❌ 没有 test_data 参数
    # ❌ 没有 connection_id 参数
)
```

**证据3**：对比同步生成方式
在同步生成方式中（`test_cases.py` 第141-147行），可以看到：
```python
test_data = generator.generate_test_data_for_api(
    api_info=api_data,
    connection_id=None,  # 即使同步生成，也不使用数据库连接
    project_id=project_id,
    use_real_data=False,  # 不使用真实数据
    db_session=db
)
```
即使同步生成，也设置了 `connection_id=None` 和 `use_real_data=False`。

---

### ✅ 只关联系统数据库（接口定义）

异步生成测试用例**只读取系统数据库中的接口定义信息**：
- ✅ 从 `APIInterface` 表读取接口定义（方法、URL、参数等）
- ✅ 从 `TestCase` 表读取测试用例记录
- ❌ **不读取业务数据库数据**（如用户表、订单表等）

---

## PytestCaseGenerator 内部逻辑

**代码位置**：`backend/app/services/test_case_generator.py` 第38-200行

### 生成流程

1. **判断是否使用LLM生成**（第64-69行）
   ```python
   use_llm_generation = use_llm if use_llm is not None else self.use_llm
   if use_llm_generation:
       return self._generate_with_llm(...)  # 使用LLM生成
   ```

2. **传统方式生成**（第71行开始）
   - 提取接口信息（name, method, url, params, headers, body）
   - 如果没有传入 `test_data` 参数，使用接口定义中的参数作为模板
   - 生成pytest测试代码结构
   - 使用 Faker 库生成模拟数据（如果需要在代码中硬编码数据）

3. **数据生成器的使用**
   - `PytestCaseGenerator` 类内部有 `self.data_generator = SmartTestDataGenerator()` 实例
   - 但在 `generate_test_case` 方法中，**只有在传入 `test_data` 参数时才会使用**
   - 异步生成时没有传入 `test_data`，所以不会调用数据生成器

## 对比：何时会使用数据库数据？

### 场景1：任务准备阶段（TaskPreparationService）

**代码位置**：`backend/app/services/task_preparation.py` 第147-154行

```python
case_test_data = self.data_generator.generate_test_data_for_api(
    api_info=api_info,
    connection_id=connection_id if connection_id else None,  # ✅ 可以使用数据库连接
    project_id=project_id,
    use_real_data=bool(connection_id and engine),  # ✅ 如果提供了连接，使用真实数据
    db_session=self.db,
    engine=engine
)
```

**使用场景**：
- 在执行测试任务之前，系统会调用 `TaskPreparationService.prepare_task()` 准备测试数据
- 此时会使用 `SmartTestDataGenerator` 生成测试数据
- 如果提供了 `connection_id` 和 `engine`，会从业务数据库提取真实数据

### 场景2：场景测试用例生成

**代码位置**：`backend/app/celery_tasks.py` `generate_scenario_test_case_task`

场景测试用例生成时，可能会使用数据库数据来生成更准确的场景流程。

## 总结

| 阶段 | 是否使用数据库数据 | 说明 |
|------|------------------|------|
| **异步生成测试用例**（`generate_test_case_task`） | ❌ **不使用** | 只基于接口定义生成测试代码结构，不生成测试数据，不使用业务数据库 |
| **任务准备阶段**（`TaskPreparationService`） | ✅ **使用** | 在执行任务前，如果提供了数据库连接，会从业务数据库提取真实数据 |
| **测试用例执行阶段** | ✅ **可能使用** | 执行时会使用任务准备阶段生成的测试数据 |

**结论**：
- ✅ 异步生成测试用例（`generate_test_case_task`）**不关联业务数据库数据**
- ✅ 只基于接口定义信息（method, url, params, headers, body）生成测试代码框架
- ✅ 测试数据在实际执行任务时，由 `TaskPreparationService` 根据数据库连接情况生成
