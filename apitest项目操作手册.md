## 项目概述

apitest是一个API接口智能测试系统，支持三种类型的测试任务：

- **接口测试任务**：针对单个接口的测试
- **场景接口测试任务**：基于场景用例集的完整业务流程测试
- **性能测试任务**：使用JMeter进行性能压测

## 前置准备工作

在创建测试任务之前，需要完成以下准备工作：

### 0. （可选但推荐）配置业务数据库连接

**路径**：左侧菜单 → `数据库信息`

**数据库的作用**：
业务数据库连接是apitest系统的一个重要功能，用于提升测试数据生成的准确性和真实性。系统通过连接业务数据库，可以：

1. **提取数据库表结构信息**：
   - 自动分析数据库中的表结构、字段类型、约束关系
   - 识别主键、外键、索引等关系
   - 提取字段注释和业务含义

2. **构建知识图谱**：
   - 将数据库表结构信息存储到Neo4j图数据库中
   - 建立表与表之间的关联关系
   - 为接口依赖分析提供数据基础

3. **关联接口与数据库表**：
   - 通过AI分析接口参数名称与数据库字段的匹配关系
   - 识别接口参数对应的数据库表和字段
   - 例如：接口参数 `user_id` 会自动关联到 `users` 表的 `id` 字段

4. **生成真实测试数据**：
   - 基于数据库中的实际数据生成测试用例
   - 确保测试数据符合数据库约束（外键、唯一性等）
   - 提高测试用例的准确性和可执行性

**什么时候使用数据库**：

- ✅ **接口依赖分析时**：系统会使用数据库信息来识别接口之间的数据依赖关系
- ✅ **生成场景用例集时**：基于数据库表关系，系统可以更准确地识别接口调用顺序
- ✅ **生成测试用例时**：系统会从数据库中提取真实数据，生成更准确的测试参数
- ✅ **执行测试任务时**：如果启用了"自动准备"，系统会使用数据库信息构造测试数据

**操作步骤**：

1. 进入 `数据库信息` 页面
2. 点击 `创建数据库连接` 按钮
3. 填写数据库连接信息：
   - **数据库类型**：选择 `MySQL`（目前主要支持MySQL）
   - **主机地址**：数据库服务器地址，例如：`localhost` 或 `120.92.88.231`
   - **端口**：数据库端口，MySQL默认是 `3306`
   - **数据库名**：要连接的数据库名称，例如：`uitest` 或 `xiaojin`
   - **用户名**：数据库用户名，例如：`root` 或 `admin`
   - **密码**：数据库密码
4. 点击 `测试连接` 按钮，确认连接成功
5. 点击 `确定` 保存连接

**注意事项**：
- 数据库连接是可选的，不配置也可以使用系统
- 但配置数据库后，测试数据生成会更准确
- 系统会提取数据库元数据，不会修改数据库内容
- 建议连接测试环境的数据库，不要连接生产环境

**数据库连接的使用场景**：

| 场景 | 是否必需 | 作用 |
|------|---------|------|
| 接口依赖分析 | 推荐 | 通过数据库表关系识别接口依赖 |
| 生成场景用例集 | 推荐 | 基于数据流关系生成更准确的场景 |
| 生成测试用例 | 推荐 | 从数据库提取真实数据作为测试参数 |
| 执行接口测试任务 | 可选 | 如果启用"自动准备"，会使用数据库构造数据 |
| 执行场景接口测试任务 | 推荐 | 场景测试需要真实数据来验证业务流程 |
| 执行性能测试任务 | 可选 | 性能测试通常使用模拟数据 |

### 1. 上传并解析接口文档

**路径**：左侧菜单 → `文档管理`

**操作步骤**：

1. 点击页面右上角的 `上传文档` 按钮
2. 选择支持的文件格式：
   - Swagger/OpenAPI（`.yaml`, `.yml`, `.json`）⭐推荐
   - Postman Collection（`.json`）
   - Apifox格式（`.json`）
   - JSON请求日志（`.json`）
   - 其他格式（`.txt`, `.md`, `.pdf`, `.docx`, `.xlsx`等）
3. 上传后系统会自动解析，等待解析完成（状态显示为"已解析"）
4. 解析完成后，接口会出现在 `接口列表` 页面

**注意事项**：
- 如果文档一直显示"解析中"，可以点击"重新解析"按钮
- 解析失败会显示为"错误"状态，可以查看错误信息

### 2. 查看接口列表

**路径**：左侧菜单 → `接口列表`

**功能**：
- 查看所有已解析的接口
- 可以编辑、删除接口
- 可以批量删除接口
- 可以分析接口依赖关系

**操作步骤**：
1. 进入接口列表页面
2. 确认需要测试的接口已存在
3. 可以查看接口详情（点击"查看"按钮）

### 3. 配置测试环境

**路径**：左侧菜单 → `测试环境`

**操作步骤**：

1. 点击页面右上角的 `添加环境` 按钮
2. 填写环境信息：
   - **环境名称**：例如"开发环境"、"测试环境"、"生产环境"
   - **Base URL**：API的基础地址，例如 `http://localhost:8000` 或 `https://api.example.com`
   - **描述**（可选）：环境的描述信息
3. 点击 `确定` 保存

**注意事项**：
- 至少需要创建一个测试环境
- Base URL 应该是完整的协议+域名+端口，例如：`http://localhost:8007`
- 创建任务时可以选择不同的环境

### 4. （可选）分析接口依赖关系

**路径**：左侧菜单 → `接口依赖分析`

**功能**：
- 分析接口之间的依赖关系
- 为生成场景用例集提供数据基础

**操作步骤**：
1. 进入接口依赖分析页面
2. 点击 `分析所有接口依赖` 按钮
3. 等待分析完成（会显示进度）
4. 分析完成后，可以查看依赖关系图

**注意事项**：
- 依赖分析需要一定时间，请耐心等待
- 分析完成后，依赖关系数据会保存，用于后续生成场景用例集
- **重要**：生成场景用例集的操作在"场景用例集"页面进行，不在"接口依赖分析"页面

### 5. （场景接口测试必需）生成场景用例集

**路径**：左侧菜单 → `场景用例集`

**前置条件**：
- 已完成接口依赖分析（见步骤4）

**操作步骤**：

**方法一：从接口依赖分析自动生成（推荐）**
1. 进入 `场景用例集` 页面
2. 点击页面上的 `从依赖分析生成小场景用例集` 按钮（通常在页面顶部）
3. 系统会自动基于依赖分析结果生成场景用例集
4. 等待生成完成（会显示进度或提示）
5. 在页面中查看生成的场景用例集

**方法二：手动创建场景用例集**
1. 在 `场景用例集` 页面，查看已有的场景用例集
2. 可以点击场景用例集卡片查看详情
3. 可以编辑场景用例集中的接口顺序
4. 可以添加或删除接口

**场景用例集说明**：
- 场景用例集定义了接口的执行顺序和依赖关系
- 一个场景用例集包含多个接口，按照顺序执行
- 场景用例集用于创建场景接口测试任务

### 6. （场景接口测试必需）从场景用例集生成测试用例

**路径**：左侧菜单 → `场景用例集`

**操作步骤**：

1. 进入 `场景用例集` 页面
2. 找到需要生成测试用例的场景用例集
3. 点击用例集卡片上的 `生成测试用例` 按钮
4. 选择用例类型：
   - **pytest（接口测试用例）**：生成pytest格式的测试用例
   - **jmeter（性能测试用例）**：生成JMeter格式的测试用例（用于性能测试）
5. 选择生成类型：
   - **场景用例**：生成场景测试用例（多个接口在一个文件中）
   - **接口用例**：为每个接口生成独立的测试用例
6. 点击 `确定` 开始生成
7. 等待生成完成（会显示进度）

**注意事项**：
- 生成测试用例需要调用DeepSeek API，需要一定时间
- 生成完成后，测试用例会出现在 `测试用例` 页面
- 场景接口测试任务需要使用场景用例类型的测试用例

### 7. （接口测试必需）查看接口测试用例

**路径**：左侧菜单 → `测试用例`

**重要说明**：
- **测试用例页面有三个标签页，对应三种不同类型的用例** ⭐
- **接口测试用例**：用于接口测试任务（普通接口测试）
- **接口场景用例**：用于场景接口测试任务（场景流程测试）
- **性能测试用例**：用于性能测试任务（JMeter压测）

**操作步骤**：

**查看现有测试用例**：
1. 进入 `测试用例` 页面
2. 在页面顶部选择标签页：
   - **接口场景用例**：场景测试用例（从场景用例集生成，用于场景接口测试任务）
   - **接口测试用例**：普通接口测试用例 ⭐（用于接口测试任务）
   - **性能测试用例**：JMeter性能测试用例（用于性能测试任务）
3. 在左侧树形结构中查看已生成的测试用例
4. 点击用例可以查看详情和代码
5. 可以编辑测试用例的代码

**三种测试用例的区别**：

| 用例类型 | 生成方式 | 用途 | 任务类型 |
|---------|---------|------|---------|
| **接口测试用例** | 自动生成或手动创建 | 测试单个接口 | 接口测试任务 |
| **接口场景用例** | 从场景用例集生成 | 测试完整业务流程 | 场景接口测试任务 |
| **性能测试用例** | 从场景用例集生成（JMeter类型） | 性能压测 | 性能测试任务 |

**测试用例的生成时机**：

1. **接口测试用例（自动生成）**：
   - 创建接口测试任务时，如果没有选择测试用例，系统会自动为接口生成测试用例
   - 系统会根据接口文档中的接口信息自动生成pytest测试代码
   - 生成完成后，测试用例会出现在"测试用例"页面的"接口测试用例"标签中
   - **注意**：这些用例是独立的，每个接口一个用例文件

2. **接口场景用例（从场景用例集生成）**：
   - 在"场景用例集"页面点击"生成接口场景用例"按钮
   - 这会生成场景类型的测试用例（多个接口在一个文件中，按顺序执行）
   - 生成完成后，测试用例会出现在"测试用例"页面的"接口场景用例"标签中
   - **注意**：这些用例是场景流程，包含多个接口的调用顺序

3. **性能测试用例（从场景用例集生成）**：
   - 在"场景用例集"页面点击"生成性能测试用例"按钮（选择JMeter类型）
   - 生成完成后，测试用例会出现在"测试用例"页面的"性能测试用例"标签中

**注意事项**：
- ✅ **接口测试任务**：可以选择"接口测试用例"，也可以不选择（系统会自动生成）
- ✅ **场景接口测试任务**：必须选择"接口场景用例"，不能选择普通接口用例
- ✅ **性能测试任务**：必须选择"性能测试用例"
- ✅ **接口测试用例和接口场景用例是不同的**：
  - 接口测试用例：每个接口独立测试，互不影响
  - 接口场景用例：多个接口按顺序执行，可以传递数据（如token）

## 接口测试任务创建流程

**适用场景**：测试单个或多个独立的接口

### 前置条件

✅ 已上传并解析接口文档  
✅ 已配置测试环境（至少一个）  
✅ **不需要预先创建测试用例**（系统会在执行任务时自动生成）

### 详细操作步骤

#### 步骤1：进入测试任务页面

1. 点击左侧菜单中的 `测试任务`
2. 页面会显示所有测试任务的列表
3. 页面顶部会显示任务类型统计：
   - 场景接口测试任务 (X)
   - 接口测试任务 (X)
   - 性能测试任务 (X)

#### 步骤2：创建新的测试任务

1. 点击页面右上角的 `添加任务` 按钮
2. 会弹出"创建测试任务"的对话框

#### 步骤3：填写任务基本信息

在对话框中填写以下信息：

1. **任务名称**：
   - 输入任务名称，例如："用户管理接口测试"

2. **任务类型**：
   - 选择 `立即执行`（立即执行）或 `定时执行`（定时执行）

3. **执行任务类型**：
   - **选择 `接口`** ⭐（这是接口测试任务的关键设置）
   - 选项包括：
     - `接口`：接口测试任务
     - `场景`：场景接口测试任务
     - `性能`：性能测试任务

4. **测试环境**：
   - 从下拉列表中选择之前配置的测试环境
   - 如果没有环境，需要先去"测试环境"页面创建

5. **用例集**（可选）：
   - 如果已经创建了用例集，可以选择
   - 也可以留空，系统会执行所有测试用例

#### 步骤4：选择测试用例（可选）

**重要说明**：
- **接口测试任务**：可以选择"接口测试用例"（普通接口用例），也可以不选择（系统会自动生成）
- **场景接口测试任务**：只能选择"接口场景用例"（场景用例），不能选择普通接口用例
- **性能测试任务**：只能选择"性能测试用例"（JMeter用例）

**操作步骤**：

1. 在对话框的"测试用例"下拉框中，系统会根据任务类型自动过滤：
   - **接口测试任务**：显示"接口测试用例"标签页中的用例（`is_scenario=false` 的pytest用例）
   - **场景接口测试任务**：显示"接口场景用例"标签页中的用例（`is_scenario=true` 的pytest用例）
   - **性能测试任务**：显示"性能测试用例"标签页中的用例（`case_type='jmeter'` 的用例）

2. **如果没有选择测试用例**：
   - **接口测试任务**：系统会在执行任务时自动为接口生成测试用例 ⭐
   - **场景接口测试任务**：必须选择场景用例，不能自动生成
   - **性能测试任务**：必须选择性能用例，不能自动生成

3. 如果下拉框为空或显示"暂无符合条件的测试用例"：
   - **接口测试任务**：这是正常的，系统会在执行时自动生成
   - **场景接口测试任务**：需要先去"场景用例集"页面生成场景用例
   - **性能测试任务**：需要先生成性能测试用例

**测试用例类型说明**：

| 用例类型 | 标签页位置 | 数据库标识 | 用途 | 如何生成 |
|---------|-----------|-----------|------|---------|
| **接口测试用例** | 测试用例 → 接口测试用例 | `case_type='pytest'`<br>`is_scenario=false` | 用于接口测试任务 | 自动生成或手动创建 |
| **接口场景用例** | 测试用例 → 接口场景用例 | `case_type='pytest'`<br>`is_scenario=true` | 用于场景接口测试任务 | 从场景用例集生成 |
| **性能测试用例** | 测试用例 → 性能测试用例 | `case_type='jmeter'` | 用于性能测试任务 | 从场景用例集生成（JMeter类型） |

**接口测试用例 vs 接口场景用例的区别**：

| 特性 | 接口测试用例 | 接口场景用例 |
|------|------------|-------------|
| **用例结构** | 每个接口一个独立的测试文件 | 多个接口在一个测试文件中，按顺序执行 |
| **数据传递** | 不传递数据，每个接口独立测试 | 可以传递数据（如token、ID等） |
| **执行方式** | 并行或独立执行 | 按顺序执行，有依赖关系 |
| **适用场景** | 测试单个接口的功能 | 测试完整的业务流程 |
| **覆盖范围** | 不覆盖所有接口，只测试选中的接口 | 覆盖场景用例集中的所有接口 |

**重要**：
- **接口测试用例和接口场景用例是不同的**，它们不会互相覆盖
- **接口测试用例**：用于测试单个接口，每个接口独立
- **接口场景用例**：用于测试完整业务流程，多个接口按顺序执行
- 如果创建"接口测试任务"，应该选择"接口测试用例"标签页中的用例
- 如果创建"场景接口测试任务"，应该选择"接口场景用例"标签页中的用例

**推荐做法**：
- **接口测试任务**：第一次可以不选择测试用例，让系统自动生成
- **场景接口测试任务**：必须先生成场景用例集，然后生成场景用例
- **性能测试任务**：必须先生成性能测试用例

#### 步骤5：高级设置（可选）

1. **自动准备**：
   - 勾选后，系统会自动准备测试数据

2. **最大重试次数**：
   - 设置任务失败时的最大重试次数（默认3次）

#### 步骤6：提交任务

1. 检查所有信息无误后
2. 点击对话框右下角的 `确定` 按钮
3. 任务会立即创建并开始执行（如果选择了"立即执行"）

#### 步骤7：查看任务执行状态

1. 任务创建后，会在任务列表中显示
2. 任务状态会显示为：
   - `pending`：等待执行
   - `running`：正在执行
   - `completed`：执行完成
   - `failed`：执行失败
   - `stopped`：已停止

3. 可以点击任务行的 `查看` 按钮查看详细执行结果
4. 可以点击 `查看报告` 按钮查看Allure测试报告

## 场景接口测试任务创建流程

**适用场景**：测试完整的业务流程，包含多个有依赖关系的接口

### 前置条件

✅ 已上传并解析接口文档  
✅ 已配置测试环境（至少一个）  
✅ 已完成接口依赖分析  
✅ 已生成场景用例集  
✅ 已从场景用例集生成测试用例（场景用例类型）

### 详细操作步骤

#### 步骤1：生成场景用例集（如果还没有）

1. 进入 `接口依赖分析` 页面
2. 点击 `分析所有接口依赖` 按钮
3. 等待分析完成
4. 点击 `生成小场景用例集` 按钮
5. 系统会自动生成场景用例集

#### 步骤2：从场景用例集生成测试用例

1. 进入 `场景用例集` 页面
2. 找到需要测试的场景用例集
3. 点击用例集卡片上的 `生成测试用例` 按钮
4. 在弹出对话框中选择：
   - **用例类型**：选择 `pytest（接口测试用例）`
   - **生成类型**：选择 `场景用例` ⭐（重要：必须是场景用例）
5. 点击 `确定` 开始生成
6. 等待生成完成（会显示生成进度）

**注意事项**：
- 必须选择"场景用例"类型，不能选择"接口用例"
- 生成完成后，测试用例会出现在"测试用例"页面

#### 步骤3：进入测试任务页面

1. 点击左侧菜单中的 `测试任务`
2. 页面会显示所有测试任务的列表

#### 步骤4：创建新的测试任务

1. 点击页面右上角的 `添加任务` 按钮
2. 会弹出"创建测试任务"的对话框

#### 步骤5：填写任务基本信息

在对话框中填写以下信息：

1. **任务名称**：
   - 输入任务名称，例如："用户注册登录场景测试"

2. **任务类型**：
   - 选择 `立即执行` 或 `定时执行`

3. **执行任务类型**：
   - **选择 `场景`** ⭐（这是场景接口测试任务的关键设置）
   - 必须选择 `场景`，不能选择 `接口` 或 `性能`

4. **测试环境**：
   - 从下拉列表中选择测试环境

5. **用例集**：
   - 选择包含场景用例的用例集
   - 或者选择场景用例集对应的测试用例

#### 步骤6：选择场景用例集

1. 在"用例集"区域，选择包含场景用例的用例集
2. 确保用例集中的测试用例是"场景用例"类型
3. 可以在"测试用例"页面确认用例类型

#### 步骤7：提交任务

1. 检查所有信息无误后
2. 点击对话框右下角的 `确定` 按钮
3. 任务会立即创建并开始执行

#### 步骤8：查看任务执行状态

1. 任务创建后，会在任务列表中显示
2. 任务类型会显示为"场景接口测试任务"
3. 可以查看执行进度和结果
4. 可以查看Allure测试报告

## 数据库连接详细说明

### 数据库的作用

apitest系统支持连接业务数据库，这是一个**可选但推荐**的功能。数据库连接主要用于提升测试数据生成的准确性和真实性。

#### 核心功能

1. **数据库元数据提取**：
   - 系统会自动连接数据库，提取所有表的结构信息
   - 包括：表名、字段名、字段类型、是否主键、是否外键、字段注释等
   - 提取的信息会存储到系统的知识库中

2. **知识图谱构建**：
   - 将数据库表结构信息存储到Neo4j图数据库中
   - 建立表与表之间的关联关系（外键关系）
   - 例如：`orders` 表通过 `user_id` 外键关联到 `users` 表

3. **接口与数据库表关联**：
   - 系统通过AI分析接口参数名称与数据库字段的匹配关系
   - 例如：接口参数 `user_id` 会自动关联到 `users` 表的 `id` 字段
   - 接口参数 `order_id` 会自动关联到 `orders` 表的 `id` 字段

4. **真实测试数据生成**：
   - 基于数据库中的实际数据生成测试用例
   - 确保测试数据符合数据库约束（外键、唯一性、非空等）
   - 提高测试用例的准确性和可执行性

#### 使用场景

**场景1：接口依赖分析**
- **何时使用**：在"接口依赖分析"页面点击"分析所有接口依赖"时
- **作用**：系统会分析接口参数，通过数据库表关系识别接口之间的数据依赖
- **示例**：如果接口A返回 `user_id`，接口B需要 `user_id` 作为参数，系统会识别出A→B的依赖关系

**场景2：生成场景用例集**
- **何时使用**：在"场景用例集"页面点击"生成用例集"时
- **作用**：基于数据库表之间的外键关系，系统可以更准确地识别接口调用顺序
- **示例**：系统识别出需要先创建用户（插入 `users` 表），才能创建订单（插入 `orders` 表）

**场景3：生成测试用例**
- **何时使用**：
  - **场景接口测试用例**：从场景用例集生成测试用例时
  - **普通接口测试用例**：在执行测试任务时自动生成（如果启用了"自动准备"）
- **作用**：系统会分析接口参数，推断需要的数据库表，然后：
  - 从数据库中提取真实数据作为测试参数（如果 `use_real_data=True`）
  - 或者根据数据库表结构生成符合约束的模拟数据（如果 `use_real_data=False`）
- **实际体现**：
  - 生成的测试用例代码中，测试参数会使用数据库中的真实值
  - 例如：`user_id=123`（从 `users` 表查询得到的真实ID）
  - 例如：`device_id=456`（从 `devices` 表查询得到的真实ID）
- **注意事项**：
  - 系统会先推断接口需要的表（通过参数名匹配，如 `user_id` → `users` 表）
  - 然后查询数据库获取真实数据（如 `SELECT id FROM users LIMIT 1`）
  - 如果数据库中没有数据，系统会使用模拟数据
  - 这个功能主要在**场景接口测试**中使用，普通接口测试可能使用模拟数据

**场景4：执行测试任务（自动准备）**
- **何时使用**：创建测试任务时，如果启用了"自动准备"选项
- **作用**：系统会分析测试用例的依赖关系，从数据库中提取或构造测试数据
- **实际体现**：
  - 在任务准备阶段，系统会分析接口参数
  - 推断需要的数据库表和字段
  - 从数据库查询真实数据填充到测试用例中
  - 例如：执行订单查询接口测试时，系统会从 `orders` 表中提取真实的订单ID
- **注意事项**：
  - 这个功能主要在**场景接口测试任务**中使用
  - 普通接口测试任务可能不会主动查询数据库，而是使用已有的测试用例数据
  - 如果数据库连接失败或没有数据，系统会使用模拟数据

#### 数据库连接配置

**配置位置**：左侧菜单 → `数据库信息`

**配置步骤**：
1. 点击 `创建数据库连接` 按钮
2. 填写数据库连接信息（系统会预填默认值，通常是 `uitest` 项目的数据库）
3. 点击 `测试连接` 确认连接成功
4. 点击 `确定` 保存

**配置说明**：
- **数据库类型**：目前主要支持 MySQL
- **主机地址**：可以是 `localhost`（本地）或远程服务器地址
- **端口**：MySQL默认是 `3306`
- **数据库名**：要连接的数据库名称（不是表名）
- **用户名和密码**：数据库的登录凭证

#### 数据库使用流程

```
配置数据库连接
    ↓
系统提取数据库元数据（表结构、字段、关系）
    ↓
存储到Neo4j知识图谱
    ↓
接口依赖分析时使用（识别接口之间的数据依赖）
    ↓
生成场景用例集时使用（确定接口调用顺序）
    ↓
生成测试用例时使用（主要场景：场景接口测试用例）
    │   ├─ 推断接口需要的表（通过参数名匹配）
    │   ├─ 查询数据库获取真实数据（如：SELECT id FROM users LIMIT 1）
    │   └─ 将真实数据填充到测试用例代码中
    ↓
执行测试任务时使用（如果启用"自动准备"，主要在场景接口测试中）
    │   ├─ 分析测试用例依赖关系
    │   ├─ 从数据库提取真实数据
    │   └─ 构造测试参数
```

**重要说明**：
- 数据库数据提取主要在**场景接口测试**中使用
- 普通接口测试可能使用模拟数据，不一定查询数据库
- 系统会先推断接口需要的表，然后查询数据库获取真实数据
- 如果数据库中没有数据或连接失败，系统会使用模拟数据

#### 注意事项

1. **数据库连接是可选的**：
   - 不配置数据库也可以使用系统
   - 但配置后，测试数据生成会更准确

2. **系统只读取数据，不修改数据库**：
   - 系统只会提取数据库元数据和查询数据
   - 不会执行 INSERT、UPDATE、DELETE 等修改操作
   - 不会影响数据库的完整性

3. **建议连接测试环境数据库**：
   - 不要连接生产环境数据库
   - 建议连接测试环境或开发环境的数据库

4. **数据库连接失败不影响基本功能**：
   - 如果数据库连接失败，系统仍可以正常工作
   - 只是测试数据生成可能不够准确

5. **一个项目可以配置多个数据库连接**：
   - 如果系统有多个数据库，可以配置多个连接
   - 系统会使用最新的连接进行数据提取

## 常见问题

### Q1: 如何知道应该创建哪种类型的测试任务？

**A**: 
- 如果只想测试单个接口，选择**接口测试任务**
- 如果要测试完整的业务流程（多个接口按顺序执行），选择**场景接口测试任务**
- 如果要做性能压测，选择**性能测试任务**

### Q2: 场景接口测试任务和接口测试任务的区别？

**A**:
- **接口测试任务**：每个接口独立测试，互不影响
- **场景接口测试任务**：多个接口按顺序执行，可以传递数据（如token），适合测试完整业务流程

### Q3: 创建场景接口测试任务时提示"没有可用的测试用例"？

**A**:
1. 确保已完成接口依赖分析
2. 确保已生成场景用例集
3. 确保已从场景用例集生成测试用例，且生成类型选择了"场景用例"
4. 在"测试用例"页面确认用例类型为"场景用例"

### Q4: 性能测试任务的并发线程数如何设置？

**A**:
- 建议从小数值开始，例如：10
- 逐步增加，观察系统响应
- 根据系统承载能力设置，避免压垮系统

### Q5: 任务执行失败怎么办？

**A**:
1. 查看任务详情中的错误信息
2. 检查测试环境配置是否正确
3. 检查接口是否可访问
4. 检查测试用例代码是否正确
5. 可以点击"重试"按钮重新执行

### Q6: 如何查看测试报告？

**A**:
1. 在测试任务列表中，找到已完成的任务
2. 点击任务行的"查看报告"按钮
3. 系统会打开Allure测试报告（接口测试和场景测试）
4. 或者查看性能分析报告（性能测试）

### Q7: 测试用例可以手动编辑吗？

**A**:
- 可以，在"测试用例"页面
- 点击用例，可以查看和编辑测试代码
- 可以手动修改代码后再执行

### Q8: 如何删除测试任务？

**A**:
1. 在测试任务列表中
2. 点击任务行的"删除"按钮
3. 确认删除
4. 注意：正在执行的任务无法删除，需要先停止

### Q9: 数据库连接是必需的吗？什么时候需要配置？

**A**:
- **不是必需的**，但强烈推荐配置
- **推荐配置的场景**：
  - 需要生成场景接口测试任务时（提高场景用例集的准确性）
  - 需要生成真实测试数据时（从数据库提取实际数据）
  - 需要分析接口依赖关系时（通过数据库表关系识别依赖）
- **可以不配置的场景**：
  - 只需要执行简单的接口测试（使用模拟数据）
  - 接口参数不依赖数据库数据
- **配置后**：系统会自动提取数据库元数据，用于提升测试数据生成的准确性

### Q10: 数据库连接后，系统会修改我的数据库吗？

**A**:
- **不会**，系统只读取数据，不会修改数据库
- 系统只会：
  - 查询数据库表结构（SHOW TABLES, DESCRIBE TABLE等）
  - 查询数据（SELECT语句，用于提取测试数据，如 `SELECT id FROM users LIMIT 1`）
- 系统不会执行：
  - INSERT、UPDATE、DELETE等修改操作
  - DROP、ALTER等结构修改操作
- 可以安全地连接测试环境数据库

### Q12: 数据库数据在测试用例中如何体现？为什么我看不到关联了数据库？

**A**:
- **数据库数据主要在场景接口测试中使用**：
  - 生成场景接口测试用例时，系统会从数据库提取真实数据
  - 例如：如果接口需要 `user_id`，系统会从 `users` 表查询真实的用户ID
  - 生成的测试代码中会包含这些真实数据，如：`user_id=123`
- **普通接口测试可能使用模拟数据**：
  - 普通接口测试用例生成时，可能不会主动查询数据库
  - 而是根据接口参数类型生成模拟数据
- **如何查看是否使用了数据库数据**：
  - 查看生成的测试用例代码，如果参数值是数据库中的真实值，说明使用了数据库
  - 如果参数值是随机生成的模拟数据，说明可能没有使用数据库
- **为什么可能看不到数据库关联**：
  - 如果数据库连接失败或没有数据，系统会使用模拟数据
  - 如果接口参数无法匹配到数据库表，系统会使用模拟数据
  - 普通接口测试可能默认使用模拟数据，不查询数据库

### Q13: 新建测试任务时为什么只能选择场景测试用例？没有单独的接口用例吗？

**A**:
- **不是的，系统有三种测试用例类型**：
  1. **接口测试用例**：用于接口测试任务（普通接口测试）
  2. **接口场景用例**：用于场景接口测试任务（场景流程测试）
  3. **性能测试用例**：用于性能测试任务（JMeter压测）
- **任务类型与用例类型的对应关系**：
  - **接口测试任务**：可以选择"接口测试用例"（`is_scenario=false`），也可以不选择（系统会自动生成）
  - **场景接口测试任务**：只能选择"接口场景用例"（`is_scenario=true`），因为需要场景流程
  - **性能测试任务**：只能选择"性能测试用例"（`case_type='jmeter'`），因为需要JMeter格式
- **为什么看起来只能选择场景用例**：
  - 系统会根据任务类型自动过滤测试用例
  - 如果创建的是"场景接口测试任务"，系统只显示"接口场景用例"标签页中的用例
  - 如果创建的是"接口测试任务"，系统会显示"接口测试用例"标签页中的用例
  - 在"测试用例"页面，可以通过切换标签页查看不同类型的用例
- **接口测试用例和接口场景用例的区别**：
  - **接口测试用例**：
    - 每个接口一个独立的测试文件
    - 每个接口独立测试，互不影响
    - 不传递数据，适合测试单个接口的功能
  - **接口场景用例**：
    - 多个接口在一个测试文件中，按顺序执行
    - 可以传递数据（如token、ID等）
    - 适合测试完整的业务流程
- **场景测试用例不覆盖所有接口用例**：
  - 场景测试用例只包含场景用例集中的接口
  - 接口测试用例是独立的，每个接口一个用例
  - 它们是不同的用例类型，不会互相覆盖
- **如何查看接口测试用例**：
  - 进入"测试用例"页面
  - 切换到"接口测试用例"标签页（不是"接口场景用例"标签页）
  - 这些用例可以用于"接口测试任务"
- **如果看不到接口测试用例**：
  - 可能还没有生成接口测试用例（系统会在创建接口测试任务时自动生成）
  - 可以先创建接口测试任务，不选择测试用例，让系统自动生成
  - 生成完成后，在"测试用例"页面的"接口测试用例"标签页中可以看到

### Q11: 数据库连接失败怎么办？

**A**:
1. 检查数据库服务器是否可访问（网络连接）
2. 检查数据库用户名和密码是否正确
3. 检查数据库端口是否正确
4. 检查数据库名是否存在
5. 如果连接失败，系统仍可以正常工作，只是测试数据生成可能不够准确
6. 可以在"数据库信息"页面点击"测试连接"按钮验证连接
