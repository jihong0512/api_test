# 知识库技术整理文档

## 一、项目使用的知识库技术概览

本项目采用**混合RAG（检索增强生成）**架构，结合多种知识库技术实现智能检索和知识管理。

### 1.1 核心技术栈

| 技术组件 | 用途 | 版本/模型 |
|---------|------|----------|
| **ChromaDB** | 向量数据库，存储文档向量表示 | >=0.4.0 |
| **通义千问 Embedding** | 文本向量化模型 | text-embedding-v3 (1536维) |
| **BM25算法** | 关键词检索 | rank-bm25==0.2.2 |
| **通义千问 Reranker** | 检索结果重排序 | qwen3-rerank |
| **Neo4j** | 知识图谱（GraphRAG） | 5.15.0 |
| **Redis** | 缓存检索结果 | 5.0.1 |

#### 3.1.4 文档存储

```python
async def add_documents(
    self,
    document_id: int,
    chunks: List[str],
    metadata_list: Optional[List[Dict[str, Any]]] = None,
    content_types: Optional[List[str]] = None
):
    """添加文档到向量数据库，支持分类存储"""
    # 在线程池中执行ChromaDB操作，避免阻塞事件循环
    def _chroma_operation():
        self._ensure_chroma_connected()
        
        # 生成向量（使用多线程加速）
        vectors = self.embed(chunks, use_threading=True)
        
        # 准备数据
        ids = [f"{document_id}_{i}" for i in range(len(chunks))]
        documents = chunks
        embeddings = vectors
        
        # 准备metadata（ChromaDB要求值必须是字符串、数字或布尔值）
        metadatas = []
        for i, meta in enumerate(metadata_list or [{}] * len(chunks)):
            metadata = {
                "document_id": str(document_id),
                "content_type": (content_types or ["text"])[i],
                **meta
            }
            # 清理metadata，复杂对象转为JSON字符串
            cleaned_metadata = {}
            for k, v in metadata.items():
                if isinstance(v, (str, int, float, bool, type(None))):
                    cleaned_metadata[k] = v
                else:
                    cleaned_metadata[k] = json.dumps(v, ensure_ascii=False)
            metadatas.append(cleaned_metadata)
        
        # 插入数据到ChromaDB
        self.collection.add(
            ids=ids,
            embeddings=embeddings,
            documents=documents,
            metadatas=metadatas
        )
    
    # 在线程池中执行，避免阻塞
    loop = asyncio.get_event_loop()
    with ThreadPoolExecutor(max_workers=1) as executor:
        success = await loop.run_in_executor(executor, _chroma_operation)
```

**支持的内容类型**:
- 文本内容（text）
- 图片描述（image）
- 表格内容（table）
- 公式内容（formula）

#### 3.1.5 混合检索（核心功能）

```python
async def search(
    self,
    query: str,
    top_k: int = 10,
    use_rerank: bool = True,
    document_id: Optional[int] = None
) -> List[Dict[str, Any]]:
    """混合检索（向量检索 + BM25 + 重排序）"""
    self._ensure_chroma_connected()
    
    # 1. 向量检索
    query_vector = self.embed([query])[0]
    
    # 构建where条件（如果指定了document_id）
    where = None
    if document_id is not None:
        where = {"document_id": str(document_id)}
    
    # 执行向量搜索（取更多候选用于重排序）
    results = self.collection.query(
        query_embeddings=[query_vector],
        n_results=top_k * 2,
        where=where
    )
    
    vector_results = []
    corpus = []
    
    # 处理向量检索结果
    if results and results.get("ids") and len(results["ids"][0]) > 0:
        ids = results["ids"][0]
        documents = results["documents"][0]
        metadatas = results["metadatas"][0]
        distances = results["distances"][0]
        
        for i, doc_id in enumerate(ids):
            # 将L2距离转换为相似度分数
            score = 1.0 / (1.0 + distances[i]) if distances[i] > 0 else 1.0
            
            vector_results.append({
                "document_id": int(metadata.get("document_id", 0)),
                "chunk_text": documents[i],
                "content_type": metadata.get("content_type", "text"),
                "metadata": metadata,
                "score": score
            })
            corpus.append(documents[i])
    
    # 2. BM25关键词检索
    if len(corpus) > 0:
        tokenized_corpus = [doc.split() for doc in corpus]
        bm25 = BM25Okapi(tokenized_corpus)
        tokenized_query = query.split()
        bm25_scores = bm25.get_scores(tokenized_query)
        
        # 合并BM25分数
        for i, result in enumerate(vector_results[:len(bm25_scores)]):
            result["bm25_score"] = float(bm25_scores[i])
            # 综合分数（向量70% + BM25 30%）
            result["final_score"] = result["score"] * 0.7 + result.get("bm25_score", 0) * 0.3
    
    # 3. 重排序（使用BGE-Reranker）
    if use_rerank and len(vector_results) > 1:
        from app.services.reranker_service import RerankerService
        reranker = RerankerService()
        texts = [r["chunk_text"] for r in vector_results]
        reranked = reranker.rerank(query, texts)
        
        # 更新排序
        reranked_results = []
        for idx, score in reranked:
            if idx < len(vector_results):
                result = vector_results[idx]
                result["rerank_score"] = score
                reranked_results.append(result)
        
        vector_results = reranked_results
        vector_results.sort(key=lambda x: x.get("rerank_score", x.get("final_score", x["score"])), reverse=True)
    else:
        vector_results.sort(key=lambda x: x.get("final_score", x["score"]), reverse=True)
    
    return vector_results[:top_k]
```

**检索流程**:
1. **向量检索**: 使用余弦相似度找到语义相似的文档
2. **BM25检索**: 使用关键词匹配补充检索结果
3. **分数融合**: 向量分数70% + BM25分数30%
4. **重排序**: 使用Reranker模型重新排序（可选）
5. **返回Top-K**: 返回最相关的K个结果

### 3.2 RerankerService（重排序服务）

**文件位置**: `backend/app/services/reranker_service.py`

**核心功能**: 使用通义千问qwen3-rerank API对检索结果进行重排序

```python
class RerankerService:
    """重排序服务，使用通义千问qwen3-rerank API"""
    
    def __init__(self):
        self.qwen_api_key = settings.QWEN_API_KEY
        self.reranker_model = settings.RERANKER_MODEL  # qwen3-rerank
        dashscope.api_key = self.qwen_api_key
    
    def rerank(
        self,
        query: str,
        texts: List[str],
        top_k: Optional[int] = None
    ) -> List[Tuple[int, float]]:
        """使用通义千问qwen3-rerank API对检索结果进行重排序"""
        from dashscope import TextReRank
        
        resp = TextReRank.call(
            model=self.reranker_model,
            query=query,
            documents=texts
        )
        
        if resp.status_code == 200:
            output = resp.get("output", {})
            results_data = output.get("results", [])
            # 格式: [{"index": 0, "relevance_score": 0.95}, ...]
            reranked = [(item["index"], item["relevance_score"]) for item in results_data]
            reranked.sort(key=lambda x: x[1], reverse=True)
            
            if top_k:
                reranked = reranked[:top_k]
            
            return reranked
```

### 3.3 HybridRAGService（混合RAG服务）

**文件位置**: `backend/app/services/rag_service.py`

**核心功能**: 整合向量检索、BM25、重排序和知识图谱检索

```python
class HybridRAGService:
    """混合RAG检索服务，使用ChromaDB + BM25 + 重排序"""
    
    def __init__(self):
        self.vector_service = VectorService()
        self.db_service = DatabaseService()  # Neo4j
        self.reranker_service = RerankerService()
    
    async def hybrid_search(
        self,
        query: str,
        project_id: int,
        top_k: int = 10,
        use_rerank: bool = True
    ) -> List[Dict[str, Any]]:
        """混合检索：向量检索 + BM25 + 重排序"""
        results = await self.vector_service.search(query, top_k, use_rerank)
        
        # 转换结果格式
        formatted_results = []
        for r in results:
            formatted_results.append({
                "text": r.get("chunk_text", ""),
                "score": r.get("final_score", r.get("score", 0)),
                "metadata": r.get("metadata", {}),
                "content_type": r.get("content_type", "text"),
                "document_id": r.get("document_id")
            })
        
        return formatted_results
    
    async def graph_rag_search(
        self,
        query: str,
        project_id: int,
        top_k: int = 10
    ) -> Dict[str, Any]:
        """GraphRAG检索：基于知识图谱的检索增强生成"""
        # 1. 从Neo4j知识图谱中检索相关实体和关系
        cypher_query = f"""
        MATCH (n)
        WHERE n.project_id = $project_id
        AND (
            n.name CONTAINS $query
            OR n.description CONTAINS $query
        )
        OPTIONAL MATCH (n)-[r]->(m)
        RETURN n, r, m
        LIMIT {top_k * 2}
        """
        
        graph_results = self.db_service.query_knowledge_graph(
            cypher_query,
            project_id
        )
        
        # 2. 从向量数据库检索相关文档
        vector_results = await self.hybrid_search(query, project_id, top_k)
        
        # 3. 融合图谱和向量检索结果
        graph_context = self._extract_graph_context(graph_results)
        vector_context = "\n".join([r["text"] for r in vector_results[:5]])
        
        return {
            "query": query,
            "graph_context": graph_context,
            "vector_context": vector_context,
            "graph_results": graph_results,
            "vector_results": vector_results
        }
    
    async def query(
        self,
        query: str,
        project_id: int,
        mode: str = "hybrid",  # hybrid, graph, graph_rag
        top_k: int = 10
    ) -> Dict[str, Any]:
        """统一查询接口"""
        if mode == "hybrid":
            results = await self.hybrid_search(query, project_id, top_k)
            return {"results": results, "mode": "hybrid"}
        
        elif mode == "graph":
            graph_results = self.db_service.query_knowledge_graph(...)
            return {"results": graph_results, "mode": "graph"}
        
        elif mode == "graph_rag":
            return await self.graph_rag_search(query, project_id, top_k)
```

**支持的检索模式**:
- `hybrid`: 混合检索（向量 + BM25 + 重排序）
- `graph`: 知识图谱检索（Neo4j）
- `graph_rag`: 图谱增强RAG（图谱 + 向量）

## 六、技术优势

### 6.1 混合检索的优势

1. **向量检索**: 理解语义，找到概念相关的文档
2. **BM25检索**: 精确匹配关键词，补充向量检索的不足
3. **重排序**: 使用深度学习模型重新排序，提高准确性

### 6.2 性能优化

1. **多线程并发**: Embedding生成使用线程池并发
2. **批量处理**: 批量调用API，减少网络开销
3. **Redis缓存**: 缓存检索结果，提高响应速度
4. **异步处理**: 使用异步IO，避免阻塞

### 6.3 可扩展性

1. **持久化存储**: ChromaDB支持持久化，数据不丢失
2. **项目隔离**: 通过project_id和document_id隔离数据
3. **多模态支持**: 支持文本、图片、表格、公式等多种内容类型

## 八、总结

本项目采用**混合RAG架构**，结合了：
- ✅ **ChromaDB** 作为向量数据库
- ✅ **通义千问 text-embedding-v3** 进行文本向量化
- ✅ **BM25算法** 进行关键词检索
- ✅ **通义千问 qwen3-rerank** 进行结果重排序
- ✅ **Neo4j** 支持知识图谱检索（GraphRAG）
- ✅ **Redis** 缓存检索结果

这种混合检索方案能够同时利用语义理解和关键词匹配的优势，为API测试用例生成提供了强大的知识检索能力。
